## 变更 3: 新增 §11 温度计设计

### 插入位置

在 §10 设计决策记录 之后新增。

### 完整内容

```markdown
## 11. 温度计设计

### 11.1 设计决策

| 决策 | 选择 | 备选方案 | 理由 |
|------|------|---------|------|
| 数据来源 | akshare + 中证指数官方 | 有知有行页面抓取 | 独立可控，不依赖第三方页面结构 |
| 计算方法 | 有知有行公开方法论复现 | 自创算法 | 与有知有行投资理念一致 |
| 加权方式 | 等权中位数 | 市值加权 | 与有知有行一致 |
| 考察周期 | 两轮完整牛熊周期（动态窗口） | 固定 10 年 | 与有知有行一致 |
| 综合方式 | PE 分位 + PB 分位各 50% | PE 单指标 | 与有知有行一致 |
| 精确度目标 | 方向一致，数值合理偏差 | 精确复现 | 有知有行未公开精确算法 |

### 11.2 核心算法

```
输入：全 A 股上市公司当日 PE、PB + 历史序列

1. 全市场 PE 等权中位数 = median(所有上市公司 PE)
2. 全市场 PB 等权中位数 = median(所有上市公司 PB)
3. PE 分位数 = percentile_rank(当日 PE 中位数, 历史 PE 中位数序列)
4. PB 分位数 = percentile_rank(当日 PB 中位数, 历史 PB 中位数序列)
5. 综合温度 = (PE 分位数 + PB 分位数) / 2
6. 估值状态 = low(≤30) / fair(30~70) / high(≥70)
```

### 11.3 温度区间定义

| 温度区间 | 范围 | 估值状态 | ValuationState | 投资含义 |
|----------|------|----------|----------------|----------|
| 低估 | 0° ~ 30° | 低估 | `low` | 可以多买 |
| 中估 | 30° ~ 70° | 中估 | `fair` | 可以少买 |
| 高估 | 70° ~ 100° | 高估 | `high` | 不买或减仓 |

### 11.4 支持的指数

| 优先级 | 指数 | 代码 | Phase |
|--------|------|------|-------|
| P0 | 万得全 A | Wind 全 A | P19-S1 |
| P0 | 沪深 300 | 000300 | P19-S2 |

| P1 | 中证消费 | 000932 | P19-S4 |

### 11.5 模块边界

| 层级 | 模块 | 职责 | 边界约束 |
|------|------|------|---------|
| UI | `cli.py` | `fund-analysis thermometer` 命令入口 | 只依赖 Service 层 |
| Service | `ThermometerService` | 编排数据获取 → 计算 → 缓存 → 输出 | 可调用 Capability，不直接访问 akshare |
| Capability | `ThermometerCalculator` | PE/PB 分位数计算、温度综合、估值状态映射 | 纯计算，无 IO |
| Capability | `ThermometerDataSource` | 从 akshare/中证指数获取 PE/PB 数据 | 只返回结构化数据，不缓存 |
| Capability | `ThermometerCache` | 历史数据缓存、增量更新 | 只负责存储，不计算 |

### 11.6 核心类型

| 类型 | 说明 |
|------|------|
| `ThermometerReading` | 单次温度读数（指数代码、PE/PB 分位、综合温度、估值状态、数据日期） |
| `ThermometerBatchResult` | 批量温度结果（多个读数 + 计算时间 + 数据来源） |
| `PePbHistory` | PE/PB 历史序列（日期、PE、PB） |
| `ValuationState` | `Literal["low", "fair", "high", "unavailable"]` |

### 11.7 与 analyze 的集成

`fund-analysis analyze` 自动调用温度计获取对应指数温度，映射为 `ValuationState`：
- 基金类型为指数基金 → 使用其跟踪指数的温度
- 基金类型为主动基金 → 使用其业绩基准对应指数的温度
- 温度数据不可用 → `ValuationState.UNAVAILABLE`，检查清单第 6 题标记为灰灯

### 11.8 免责标注

CLI 和报告输出中必须包含：

```
⚠️ 本温度计基于有知有行公开方法论独立计算，非有知有行官方数据。
计算方法：全市场等权 PE/PB 中位数历史分位数综合。
与有知有行官方温度计可能存在合理偏差，仅供投资参考。
```

### 11.9 非目标

- 不追求与有知有行温度计数值精确一致（算法未完全公开）
- 不提供短期择时信号（温度计用于大周期判断）
- 不支持非 A 股市场温度计算（QDII 基金使用 `unavailable`）
- 不预测未来市场走势
```
