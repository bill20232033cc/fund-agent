## Finding Closure Verification

### Finding 1 (source-truth admission 条件不足) — CLOSED

Original finding: `candidate_boundary is None` is not sufficient to prove source truth. The plan lacked a positive source-truth admission contract, and any stub with `candidate_boundary=None` could be mistaken for a source-truth input.

Fixed plan changes:

- **New `FundDisclosureSourceTruthAdmissionProof` dataclass** (line 113-138): frozen, slots=True, with 11 fields:
  - `proof_kind: Literal["repository_loaded_annual_report_identity.v1"]`
  - `source_boundary: Literal["annual_report"]`
  - `fund_code: str`, `report_year: int`, `document_kind: Literal["annual_report"]`
  - `intermediate_kind: Literal["fund_disclosure_document.v1"]`
  - `source_kind: Literal["annual_report"]`
  - `repository_identity_verified: Literal[True]`
  - `source_provenance_verified: Literal[True]`
  - `locator_identity_verified: Literal[True]`
  - `parser_integrity_verified: Literal[True]`
  - `producer: Literal["FundDocumentRepository"]`
- **Protocol extension** (line 141-145): `source_truth_admission: FundDisclosureSourceTruthAdmissionProof | None` on `FundDisclosureDocumentContentIntermediate` only.
- **Admission rule** (line 151-184): `_validate_source_truth_admission()` requires ALL of: `isinstance(content intermediate)`, `candidate_boundary is None`, `failure_class is None`, `source_provenance is not None`, proof not None, and proof identity fields exactly match dispatch/intermediate identity. Missing proof → `source_truth_admission_missing`. Identity mismatch → `source_truth_admission_invalid`.
- **New gap codes**: `source_truth_admission_missing`, `source_truth_admission_invalid`.
- **New source boundary**: `source_truth_unverified`.
- **Negative test** (line 402-404): proof-missing fixture → no public values, `source_truth_admission_missing` gap present.
- **Positive test** (line 408-411): proof-positive fixture reaches extraction decision point.

Evidence: The fixed plan now has an explicit positive admission contract with 6 literal identity fields and 4 boolean attestations, cross-validated against dispatch/intermediate identity. `candidate_boundary is None` alone is no longer sufficient — it is only one of many conjunctive conditions. A stub without `source_truth_admission` fails closed.

**Finding 1 is closed.**

### Finding 2 (product_essence.v1 schema 与 key/value 提取规则) — CLOSED

Original finding: the plan lacked exact `product_essence.v1.value` keys, value types, required/optional fields, table/cell extraction rules, ambiguity handling, and exact fixture/test assertions. Implementation agent would have to invent ad hoc schema and table traversal.

Fixed plan changes:

- **Exact value shape** (line 189-277):
  - `basic_identity`: 8 fields with types, 2 required (`fund_name`, `fund_code`), 6 optional, 2 classification keys (`classified_fund_type`, `classification_basis`).
  - `product_profile`: 4 fields with types, at-least-one-of-three required.
  - `benchmark`: single `benchmark_text: str`.
  - `risk_characteristic_text`: 5 fields with exact types, `schema_version: Literal["risk_characteristic_text.v1"]`, `source_anchors: list[dict[str, object]]` with exact sub-keys.
  - Field-family status semantics: `accepted` (all four), `partial` (at least one), `missing` (none), `blocked` (admission failures only).
- **Label map** (line 283-301): 14 output paths with Chinese label tokens.
- **Table/cell extraction rules** (line 303-321):
  - Only `locator_stability == "stable"` cells.
  - Label matches `row_label_path` first, then `column_header_path`.
  - `cell_text_normalized` preferred; fallback to `cell_text`.
  - Anti-extract guards: cell text equal to label, or `项目`/`指标`/`名称`/`内容`/`说明`.
  - `fund_code` must equal dispatch context.
  - Priority: table/cell before paragraph.
  - Single value → use it. Same value duplicates → deduplicate. Different values → `ambiguous_table_or_locator`.
- **Paragraph fallback rules** (line 323-341):
  - Only allowed for 6 descriptive output paths; not for `basic_identity.*`.
  - Matching: heading_path contains label, or text starts with `{label}:`/`{label}：`/`{label} `.
  - Extract text after separator; guard against extracting just the label.
- **Anchor rules** (line 345-358): exact `source_kind`, `document_year`, `section_id`, `table_id`, `row_locator` format for table/cell vs paragraph.
- **Missing/ambiguity behavior** (line 360-367): optional missing → no gap. Required missing → `field_family_partial`. All missing → `field_family_missing`. Ambiguity → omit path + `ambiguous_table_or_locator`.
- **Test fixtures** (line 442-469): exact cell fixtures with `row_label_path` and `cell_text_normalized`, exact assertions on keys, values, `extraction_mode`, `anchors`, `candidate_evidence`, gap codes. Includes positive exact-shape, proof-required, ambiguity, paragraph fallback, all-missing, and partial test cases.

Evidence: The fixed plan defines the exact `product_essence.v1.value` dict shape with typed keys, required/optional semantics, a label map for 14 output paths, table/cell extraction priority and ambiguity rules, paragraph fallback with allowed/blocked paths, and explicit test fixtures with exact assertions. The `FundDisclosureCellLike` protocol has no sibling reference — the plan never requires sibling lookup, instead using `row_label_path` as the key indicator with `cell_text_normalized` as the value. The plan says "Slice B must not read sibling cells" (line 306).

**Finding 2 is closed.**
