## Findings

### PR-1-未修复-高-live command lacks an authorized projection source

- **位置**: `Authorized Live Evidence Command`; `Contract`; `Goal`; `Affected Files`
- **问题类型**: 不可直接实施 / 架构边界 / 测试缺口
- **当前写法**: Plan defines `EvidenceConfirmRepositoryRunRequest.projection: ChapterFactProjection`, and the runner must call `build_annual_report_evidence_confirm_references()` followed by optional `confirm_projection_evidence_v2()`. The live command is only `uv run python scripts/evidence_confirm_ec_p2_live_sample.py --fund-code 004393 --report-year 2025 --force-refresh`.
- **反例/失败场景**: Implementation worker writes the live script and has no plan-approved way to create a `ChapterFactProjection` for `004393 / 2025`. To make the command run, the worker must invent one of these unreviewed choices: call extraction/facade logic, call Service, use a synthetic projection from another fund/year, hand-code anchors, or skip V2. Each choice changes what the live evidence proves.
- **为什么有问题**: EC-P2 is supposed to prove repository-bounded live source/PDF entry into the EC-P1A materializer. Without an authorized projection source for the live sample, the implementation cannot prove the materializer over the live sample without either broadening scope or weakening the live evidence to repository load only. The plan therefore is not code-generation-ready.
- **直接证据**:
  - Plan requires `projection: ChapterFactProjection` in `EvidenceConfirmRepositoryRunRequest`.
  - Plan live command provides only `fund-code`, `report-year`, and `force-refresh`.
  - `build_annual_report_evidence_confirm_references()` requires a caller-supplied projection and only materializes anchors already present in that projection.
  - Current EC-P2 non-goals exclude Service/UI/renderer/quality-gate integration and deterministic production facade composition beyond the repository runner.
- **影响**: Implementation Agent 跑偏 / review 不可验收 / live evidence 口径不清 / 后续返工。
- **建议改法和验证点**:
  - Amend plan to specify the exact live projection source before implementation.
  - Acceptable narrow options must be explicit, for example:
    - EC-P2 live command only proves repository load + metadata admission + materializer callable with a checked-in minimal EC-P2 projection fixture for `004393 / 2025`, and records that fact correctness is not proven; or
    - EC-P2 explicitly uses a Fund-layer projection path already in scope, with exact modules, inputs, and tests, if that is intended.
  - The live evidence artifact must state whether V2 was run against a real projected fact, a reviewed fixture projection, or not run.
  - Add tests proving the live script rejects missing projection input rather than inventing anchors.
- **修复风险（低/中/高）**: 中
- **严重程度（低/中/高/严重）**: 高

### PR-2-未修复-中-write set and existing import isolation test conflict with planned runner/script

- **位置**: `Affected Files`; `Authorized Live Evidence Command`; `Validation Commands`; existing `tests/fund/test_evidence_confirm_sources.py::test_import_isolation_does_not_import_repository`
- **问题类型**: 范围漂移 / 测试缺口 / 不可直接实施
- **当前写法**: Plan allows implementation edits in `fund_agent/fund/evidence_confirm_sources.py`, `tests/fund/test_evidence_confirm_sources.py`, READMEs and `docs/reviews/`. It also says a script may be added at `scripts/evidence_confirm_ec_p2_live_sample.py`. Existing import isolation test asserts the source file does not contain the string `"load_annual_report"`.
- **反例/失败场景**: Worker follows the plan and adds `scripts/evidence_confirm_ec_p2_live_sample.py`, but `scripts/` is not in the allowed implementation write set or `git diff --check` path list. Worker also adds the planned runner in `evidence_confirm_sources.py`; the existing import isolation test fails because the source now necessarily contains `"load_annual_report"`, even if the import remains lazy and safe.
- **为什么有问题**: A code-generation-ready plan must not require unlisted file writes or leave known existing tests contradicting the intended implementation. This forces the implementation worker to decide whether to violate the write set, skip the script, weaken the test, or redesign the runner placement.
- **直接证据**:
  - Plan live command path is `scripts/evidence_confirm_ec_p2_live_sample.py`.
  - Plan allowed files do not include `scripts/evidence_confirm_ec_p2_live_sample.py`.
  - Plan validation `git diff --check` does not include the script path.
  - Existing test `test_import_isolation_does_not_import_repository()` asserts `"load_annual_report" not in source`.
  - The planned runner must call `repository.load_annual_report(...)`.
- **影响**: Implementation Agent 跑偏 / CI failure / review 不可验收。
- **建议改法和验证点**:
  - Either add `scripts/evidence_confirm_ec_p2_live_sample.py` to allowed files and validation, or remove the script requirement and specify the inline reviewed execution snippet as the only allowed live path.
  - Amend import isolation expectations: keep the guarantee that module import does not instantiate repository or touch network/cache/PDF, but no longer assert that the source text lacks `"load_annual_report"` once EC-P2 adds the repository runner.
  - Add a replacement isolation test that monkeypatches `FundDocumentRepository.__init__`, imports `evidence_confirm_sources`, and only fails if import-time instantiation occurs.
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 中
