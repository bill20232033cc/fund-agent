### 001-未修复-高-V2 对部分悬空 `evidence_anchor_ids` 误报 hard gate pass

- **入口/函数**: `confirm_chapter_evidence_v2()` -> `_confirm_fact_v2()` -> `_dimension_missing_evidence()` / `_dimension_proof_boundary()`
- **文件(行号)**: `fund_agent/fund/evidence_confirm.py:501`, `fund_agent/fund/evidence_confirm.py:828`, `fund_agent/fund/evidence_confirm.py:898`
- **输入场景**: 一个 `available`、非 derived fact 同时带有一个有效 anchor 和一个悬空 anchor，例如 `fact.evidence_anchor_ids == ("anchor:valid", "anchor:missing")`；调用方只为有效 anchor 传入 proven same-anchor reference，且该 excerpt 能匹配 fact value。
- **实际分支**: `_confirm_fact_v2()` 只在 `not known_anchors` 时 fail，因此只要至少一个 anchor 存在就越过 E3 blocking 分支；`_dimension_missing_evidence()` 只看 `proof_references` 非空并直接 `pass`；`_dimension_proof_boundary()` 在没有 `all_fact_references` 时才 not_applicable，并且只遍历实际传入的 references，不遍历 `fact.evidence_anchor_ids`，所以未传 reference 的悬空 anchor 不会进入任何 issue。
- **预期行为**: accepted V2 plan 明确要求 `missing_evidence` 对 `dangling anchors` fail，并触发 hard gate fail；一个 fact 只要声明了当前章节不存在的 anchor id，就不能被整体复核为 `pass`。
- **实际行为**: 该 fact 的五个维度均可返回 `pass`，fact hard gate、aggregate hard gate 和 `overall_status` 都是 `pass`，`auditability_score == 100`，且 `issues == ()`。
- **直接证据**:
  - `fund_agent/fund/evidence_confirm.py:501-504` 只处理全部 anchor 都不存在的情况，未检查 `len(known_anchors) != len(fact.evidence_anchor_ids)`。
  - `fund_agent/fund/evidence_confirm.py:828-839` 在存在任一 proven reference 时把 `missing_evidence` 置为 `pass`，未验证全部 declared anchors 都 known。
  - `fund_agent/fund/evidence_confirm.py:898-927` 的 proof-boundary 只检查 `all_fact_references`，而 `all_fact_references` 来自 `references_by_anchor.get(anchor_id)`；悬空 anchor 如果没有 reference，就不会被检查。
  - `docs/reviews/evidence-confirm-scoring-v2-plan-20260621.md:182-183` 把 `dangling anchors` 列为 `missing_evidence` fail 条件。
  - 最小复现命令返回 `pass pass 100` 且 issues 为空：

```bash
uv run python -c "from dataclasses import replace; from tests.fund.test_evidence_confirm import _chapter_and_fact, _reference; from fund_agent.fund.evidence_confirm import confirm_chapter_evidence_v2; chapter,fact=_chapter_and_fact('structured.turnover_rate'); fact2=replace(fact,evidence_anchor_ids=(fact.evidence_anchor_ids[0], 'anchor:missing')); chapter2=replace(chapter,facts=(fact2,)); result=confirm_chapter_evidence_v2(chapter2, (_reference(fact.evidence_anchor_ids[0], excerpt_text='年报披露换手率为 120%。'),)); print(result.overall_status, result.fact_results[0].status, result.fact_results[0].auditability_score); print(result.issues)"
```

- **影响**: V2 hard gate 可以对带悬空 anchor id 的 fact 给出 pass-like 结果和满分，削弱 `missing_evidence` / proof-boundary 的 fail-closed contract。后续若 report-quality bridge 或人工复核依赖 V2 hard gate，会把不完整或错误的 anchor linkage 误判为可审计通过。
- **建议改法和验证点**: 在 `_confirm_fact_v2()` 进入维度计算前或在 `_dimension_missing_evidence()` 中显式计算 `dangling_anchor_ids = tuple(id for id in fact.evidence_anchor_ids if id not in anchors_by_id)`；有任一 dangling id 时产生 E3 blocking issue，`missing_evidence` fail、fact hard gate fail、fact score 0。新增 regression：一个有效 anchor + 一个 dangling anchor + 有效 proven reference 时，`overall_status == "fail"`，`missing_evidence.status == "fail"`，`blocking_issue_ids` 非空，`auditability_score == 0` 或至少不超过 blocking cap。
- **修复风险（低/中/高）**: 低。变更局限于 V2 no-live Evidence Confirm 的 declared-anchor 完整性校验；V1 行为可保持不变，已有 pass/mixed-reference/value-match tests 不应受影响。
- **严重程度（低/中/高/严重）**: 高。
