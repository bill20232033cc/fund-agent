## Findings

### PR-001-未修复-高-V2 token/matcher 同源性未被写成硬约束

- **位置**: §6 Planned Diagnostic Design, §7 Slice A2-S1
- **问题类型**: 契约缺失 / 不可直接实施 / 测试缺口
- **当前写法**: Plan 允许 helper “Internally reproduce or call existing V2 normalization/token logic only through safe abstractions. If existing helper reuse would require exposing private functions broadly, duplicate small diagnostic-only categorization logic rather than changing V2 matching semantics.”
- **反例/失败场景**: Implementation agent 为了避免触碰 `evidence_confirm.py`，在新 helper 中复制一套“相似”的 flatten/normalize/categorize 逻辑。该逻辑与 `_material_tokens()`、`_flatten_material_values()`、`_token_matches_excerpt()` 在 ignored keys、dataclass/dict/list flatten 顺序、numeric percent handling 或 substring normalization 上轻微不同。A2 evidence 随后把 live R1-R4 归类为 `value_shape_overbroad` 或 `coarse_reference_insufficient`，但这个分类并不是当前 V2 真实失败路径的结果。
- **为什么有问题**: A2 的目标是解释 deterministic V2 当前 fail。若诊断 helper 不与 V2 token/matcher 同源，诊断输出会变成第二套近似规则，不能作为下一 fix gate 的真源。用户目标要求证据与结论一致，不能用间接或近似证据判断 root cause。
- **直接证据**:
  - `fund_agent/fund/evidence_confirm.py` 的 `_dimension_value_match()` 直接调用 `_material_tokens(fact.value)` 和 `_matched_anchor_ids(tokens, proof_references)` 判定 fail/pass。
  - `_material_tokens()` 依赖 `_flatten_material_values()`，其中 dict key 排序、`_IGNORED_VALUE_KEYS`、dataclass `asdict()`、list/tuple recursion 都会影响 token set。
  - `_token_matches_excerpt()` 对 numeric token 和 text token 使用不同逻辑，numeric 又要求百分号单位相容。
  - Plan 当前没有要求 helper 输出与 V2 当前 value-match 判定的一致性测试，只要求分类测试。
- **影响**: A2 evidence 可能看似完成，但后续 fix plan 会基于错误 root-cause 分类，导致修错层、放大 scope 或错误修改 V2/Extractor。
- **建议改法和验证点**:
  - Plan 必须把“V2 同源诊断”写成硬约束：A2 helper 只能使用从 `evidence_confirm.py` 提取出的内部共享 token/matcher primitives，或通过同模块受控 diagnostic wrapper 调用现有私有函数；不得实现一套独立近似 matcher。
  - 如果必须新增 helper，先在 `evidence_confirm.py` 暴露 package-private diagnostic primitive，保持产品 public API 不变。
  - 测试必须覆盖“diagnostic matched/unmatched category 与 `confirm_projection_evidence_v2()` 的 `value_match` pass/fail 结果一致”的 golden synthetic cases：text match、numeric decimal equivalence、percent mismatch、ignored key、dataclass/list/dict nested value。
  - A2 evidence artifact 必须声明 diagnostic output is derived from V2 token/matcher primitives, not a parallel matcher.
- **修复风险（低/中/高）**: 中
- **严重程度（低/中/高/严重）**: 高
