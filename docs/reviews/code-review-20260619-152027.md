### 4. current_stage.v1 完全未接入

### 5. S6-B/C/D/E 既有 selector 语义未被破坏

`_field_families_for_intermediate()` 重构使用 mapping 替代嵌套 ternary，但对 S6-B/C/D/E 调用的 selector 函数和传参完全不变。`candidate_evidence_by_family` 的 5 个 value 与原 5 个局部变量一一对应。测试 `test_core_risk_selector_preserves_overlap_family_semantics` 用 baseline 对比法确认 S6-B/C/D/E 的 record count、path order、gap、value、anchors 均未改变。全部 79 个测试通过。

### 6. _field_families_for_intermediate 使用 mapping 而非嵌套 ternary

行 549-566：新增 `candidate_evidence_by_family` dict mapping，迭代 `_FAMILY_ORDER` 并用 `.get(family_id)` 选择路径。无第六层嵌套 conditional expression。`current_stage.v1` 不在 mapping 中，保持 `field_family_missing`。

### 7. core-risk cell generic guard role-invariant 且排除 cell_text/cell_text_normalized

`_core_risk_cell_guard_context()` 返回 parent table heading/caption/path + cell row_label_path/column_header_path/heading_path，不含 `cell_text` 或 `cell_text_normalized`。测试 `test_core_risk_selector_blocks_cell_self_guard_for_broad_tokens` 确认 cell 含 "基金产品资料概要提示风险"（broad token "风险" + guard-looking "基金产品资料概要"）但 parent table 无 guard context 时被阻断；parent table heading 含 guard token "基金简介" 时允许通过。

`_CORE_RISK_MATCH_GROUPS` 不含 `压力测试`、`最大回撤`、`否决`、`一票否决`、`最致命`、`需要替换`、`值得持有`、`风险等级`，也不含 S6-E-owned `基金份额变动`、`申购`、`赎回`、`净申购`、`净赎回`、`收益分配`、`分红`、`红利`。测试 10 和 11 分别验证 forbidden-only content 产生空 evidence，mixed forbidden+legal content 只产生 legal token 的 record。
