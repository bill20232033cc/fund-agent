以下为非阻断观察项：

### 观察 1: `source_provenance` 校验顺序与计划微偏（非阻断）

计划要求当 `source_provenance is None` 时在 processor 前 fail-closed。实现中 `_extract_active_fund_disclosure_via_processor()` 将 `source_provenance is None` 检查放在 `processor.extract()` 之后（`data_extractor.py:494`）。运行时行为正确：S4 `FundDisclosureDocumentProcessor` 对 None provenance 返回 `contract_status="blocked"` + `gaps="source_provenance_unsafe"`，facade 随后 raise RuntimeError。当前测试 `test_explicit_disclosure_missing_provenance_fails_closed` 使用 `match="source_provenance_unsafe"` 匹配了 processor blocked 消息而非 facade 的 `"FundDisclosureDocument source_provenance is required"` 消息——测试通过纯属巧合（processor 先阻断）。若未来 processor 允许 None provenance 通过，测试将因错误消息不匹配而失败。这是测试脆弱性，非运行时缺陷。

### 观察 2: `_validate_disclosure_intermediate_identity` docstring 模板引用不精确（非阻断）

`data_extractor.py:671` docstring 引用"见模板第 1 章'产品本质'"。该函数做的是 FDD 中间态四字段身份校验（fund_code、report_year、document_kind、intermediate_kind），与模板第 1 章"产品本质"语义不直接相关。AGENTS.md 要求基金分析代码引用模板章节，但此函数是纯协议身份守卫，引用可省略或改为更准确的说明。

### 观察 3: README 变更准确（确认）

README diff 仅在 `FundDataExtractor.extract()` 段落新增 S5 显式 opt-in facade 路由描述，准确反映代码行为。未提前声明 source truth、parser replacement、golden/readiness 或 release。candidate-only / `not_proven` / `NOT_READY` 边界描述完整保留。
