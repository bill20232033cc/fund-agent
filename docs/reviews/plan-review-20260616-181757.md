## Findings

### 1-未修复-高-source-body row matching contract is under-specified

- **位置**: `## 5. Evidence Method`, `## 6. Required Matrix Fields`, `## 7. Acceptance Criteria`
- **问题类型**: 契约缺失 / 不可直接实施 / 测试缺口
- **当前写法**: The plan says to compare candidate value/excerpt to repository-loaded source body using same-report, same-section, same-page or same-table evidence when available, and classify rows as `source_body_match`, `source_body_mismatch`, `repository_source_body_unavailable`, `insufficient_public_locator`, or `semantic_assignment_residual`.
- **反例/失败场景**: An evidence worker can search `ParsedAnnualReport.raw_text`, `get_section_text()`, or `ParsedTable.rows` with different normalization and duplicate-match rules. For the same row, one implementation might count a repeated value as `source_body_match`; another might classify it as ambiguous. Numeric fields, repeated portfolio totals, repeated benchmark wording, and shared candidate locators can therefore produce non-reproducible source-truth results.
- **为什么有问题**: Source-truth evidence must be logic/data same-source and deterministic. A plan that leaves row matching, normalization, duplicate handling and proof level open forces the implementation worker to redesign the evidence contract during execution.
- **直接证据**:
  - Plan section 5 only says "same-report, same-section, same-page or same-table evidence when available" and does not define precedence or ambiguity handling.
  - `ParsedAnnualReport` public contract exposes `raw_text`, `sections`, `tables`, `metadata`, and `get_section_text()`.
  - `ParsedTable` exposes `page_number`, `table_index`, `headers`, and `rows`, but no bbox or cell locator identity.
- **影响**: Evidence could overcount source truth, hide ambiguous repeated matches, or make `SOURCE_TRUTH_EVIDENCE_ACCEPTED_FOR_SELECTED_ROWS_NOT_READY` non-reproducible. Later baseline qualification would inherit a weak proof.
- **建议改法和验证点**:
  - Add a deterministic row matching contract before execution:
    - table-backed rows must first match repository `ParsedTable` by `page_number` and normalized row/cell content;
    - text-backed rows must first match `get_section_text(section_id)` when section exists, otherwise `raw_text`;
    - duplicate normalized matches must become `ambiguous_source_body_match`, not accepted source truth;
    - candidate-only locator reuse groups must remain residual unless repository source body can independently support each field;
    - bbox-level truth is out of scope because `ParsedAnnualReport` does not expose bbox.
  - Add matrix fields for `match_strategy`, `normalized_candidate`, `normalized_source_match_count`, and `proof_level`.
  - Add acceptance criteria requiring exact one unambiguous source-body match for `source_body_match`.
- **修复风险（低/中/高）**: 中
- **严重程度（低/中/高/严重）**: 高
