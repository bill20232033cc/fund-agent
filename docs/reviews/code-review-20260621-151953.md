### 1-未修复-低-material token 归一化可能产生跨值误匹配

- **入口/函数**: `_material_tokens` → `_normalize_text`
- **文件(行号)**: `evidence_confirm.py:478-495`, `evidence_confirm.py:425-439`
- **输入场景**: fact value 为 `"1.20%"`，excerpt 包含 `"换手率120%"`
- **实际分支**: `_normalize_text` 先 NFKC + casefold，再删除全部标点（`_PUNCTUATION_RE`），再删除空白
- **预期行为**: 不同数值 token 不应互相匹配
- **实际行为**: `"1.20%"` 归一化为 `"120"`，`"120%"` 归一化为 `"120"`，token 相同导致 E2 误通过
- **直接证据**: `_PUNCTUATION_RE = re.compile(r"[，,。；;：:、（）()\[\]{}]")` 删除 `.` 和 `%`；`_WHITESPACE_RE` 删除空白；`_matched_anchor_ids` 用 `token in normalized_excerpt` 做子串匹配（`evidence_confirm.py:420`）
- **影响**: 当同一 anchor excerpt 包含多个相似数值时，可能产生 E2 误匹配。概率低：需同一 anchor excerpt 同时包含不同含义的相似数值，且 fact value 恰好归一化为相同 token。anchor-scoped matching 大幅降低误命中面。
- **建议改法和验证点**: 对 material token 增加最小长度或数字精度校验；或保留 `%` 等语义标点不删除。验证：构造 fact value `"1.20%"` + excerpt `"费率1.20%，换手率120%"` 测试用例，确认 E2 不误通过。
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 低

## Residual Risk

1. **多 anchor 部分覆盖无测试**: 当前所有测试 fixture 的 fact 恰好只有 1 个 anchor。fact 拥有 2+ anchor 但只有部分 anchor 有 proven reference 的场景未覆盖。代码行为已通过手动验证确认：部分覆盖即可通过 E2（正确行为），但缺少回归保护。
2. **E1+E2 共存场景无测试**: 单个 fact 同时触发 E1（精度不足）和 E2（值不匹配）时，fact status 为 "fail"（E2 阻断），issue_ids 同时包含 E1 和 E2。此路径未被测试覆盖。
3. **`_flatten_material_values` 递归深度无保护**: 对极深嵌套的 dataclass/dict/list 结构无递归深度限制。当前 `ChapterFactEntry.value` 的实际结构是浅层 scalar/dict/list，风险极低。
4. **`_issue` 使用 SHA1 截断 12 字符**: issue_id 碰撞概率极低（12 hex = 48 bit），但非零。当前 issue 数量级（每章 <20）不构成实际风险。
5. **phase 1 证明边界**: 该 helper 只验证调用方显式传入的 same-anchor excerpt，不验证 excerpt 是否来自真实年报。这是设计意图（no-live），但需在后续 gate 接入时明确信任边界。
