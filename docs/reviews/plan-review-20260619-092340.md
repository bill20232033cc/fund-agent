## Findings

No blocking findings.

### 001-未修复-[低]-numeric-looking excerpts can be mistaken for future field facts
- **位置**: `2. Exact Non-goals`, `4. Source Path, Ordering, Dedupe, Limit, Excerpt`, `10. Residual Risks`
- **问题类型**: 测试缺口 / 残余风险
- **当前写法**: Plan allows candidate excerpts from paragraph and cell text while explicitly keeping excerpts out of public `value` and `anchors`.
- **反例/失败场景**: A matched performance or fee cell excerpt may contain numbers that look like final return, fee, or benchmark values.
- **为什么有问题**: S6-C is only a locator gate. Numeric-looking excerpts are safe only if implementation proves they remain candidate previews and cannot be consumed as field facts.
- **直接证据**: Plan sections `2`, `4`, `7`, and `10` prohibit value extraction, public anchors, `partial` / `accepted`, and require no value leak tests.
- **影响**: Future implementation could accidentally parse or copy excerpt text into `value`, turning candidate evidence into unreviewed field correctness.
- **建议改法和验证点**: Implementation tests must assert every `return_attribution.v1` record remains absent from `value`, `anchors==()`, public status remains `missing`, and candidate records preserve `not_proven` / `not_ready`.
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 低

### 002-未修复-[低]-optional shared selector helper can accidentally alter S6-B ordering
- **位置**: `Implementation Write Set`
- **问题类型**: 过度耦合 / 回归风险
- **当前写法**: Plan allows either return-attribution-specific helpers or a small private shared selector helper.
- **反例/失败场景**: A shared helper could reorder section/paragraph/table/cell traversal or change product-essence source path formatting while adding return-attribution support.
- **为什么有问题**: S6-B was accepted with binding source path and ordering semantics; S6-C should not reopen that accepted behavior.
- **直接证据**: S6-B tests assert paths such as `sections[0]`, `table_blocks[0]`, and `table_blocks[0].cells[0]`; S6-C plan requires S6-B behavior to remain unchanged.
- **影响**: Implementation could pass new return-attribution tests while subtly regressing accepted S6-B evidence identity.
- **建议改法和验证点**: If implementation introduces shared helper code, retain or strengthen S6-B regression assertions for source paths, candidate boundaries, public-empty `value`, and empty public anchors.
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 低
