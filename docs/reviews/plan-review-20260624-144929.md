## Findings

### 001-未修复-高-S1 omits transitive summary consumers and fixtures required by summary v2

- **位置**: `Implementation Slices / Slice S1: Summary V2 Provenance Contract`
- **问题类型**: 测试缺口 / 不可直接实施 / 契约缺失
- **当前写法**: S1 allowed files are only `fund_agent/fund/evidence_confirm_production.py`, `fund_agent/services/fund_analysis_service.py`, and `tests/fund/test_evidence_confirm_production.py`; validation runs only `uv run pytest tests/fund/test_evidence_confirm_production.py`.
- **反例/失败场景**: Existing Service and semantic tests call `summary_from_repository_result()` through shared fake repository results that set `reference_build_result=None`. The new contract says fact-level provenance passes only when `source_support` and `missing_evidence` pass and matched anchors map to section-or-better proven references. Those existing fixtures cannot satisfy the new provenance contract without updating their fake reference build result and V2 dimensions.
- **为什么有问题**: Implementation agent cannot both obey the S1 allowed files and keep existing Service/semantic tests coherent. If it does not update the transitive test fixtures, summary v2 can either report `provenance_status=fail` for tests that still assert successful Evidence Confirm summary, or the implementation must add defensive behavior that weakens the accepted contract by treating V2 pass without references as provenance pass.
- **直接证据**:
  - Plan S1 allowed files exclude `tests/services/test_fund_analysis_service.py` and `tests/fund/test_evidence_confirm_semantic.py`.
  - `tests/services/test_fund_analysis_service.py` helper `_repository_run_result()` constructs `EvidenceConfirmRepositoryRunResult(..., pathway_status="pass", reference_build_result=None, evidence_confirm_result=_v2_result(...))`.
  - The same file asserts product behavior from that summary, including `result.evidence_confirm_summary.status == "pass"` for pass fixtures.
  - `tests/fund/test_evidence_confirm_semantic.py` imports `_repository_run_result` from `tests.services.test_fund_analysis_service` and injects semantic results through `summary_from_repository_result()`.
  - The plan's fact-level provenance pass condition requires `at least one matched anchor maps to a proven reference whose tier is section, table or row`.
- **影响**: 实施 Agent 需要越过 plan 文件边界修改测试，或留下 failing tests，或为测试 fixture 引入与产品契约不一致的 fallback。任何一种都会使 plan 不是 code-generation-ready。
- **建议改法和验证点**:
  - Add `tests/services/test_fund_analysis_service.py` and `tests/fund/test_evidence_confirm_semantic.py` to S1 allowed files.
  - Require the shared `_repository_run_result()` / `_v2_result()` fixtures to build minimal section-level `EvidenceConfirmReferenceBuildResult` and V2 dimension results for pass/warn/fail cases.
  - Add validation:
    ```bash
    uv run pytest tests/fund/test_evidence_confirm_production.py tests/services/test_fund_analysis_service.py tests/fund/test_evidence_confirm_semantic.py
    ```
  - Keep the contract strict: no `reference_build_result=None` + V2 pass shortcut may be treated as provenance pass.
- **修复风险（低/中/高）**: 中
- **严重程度（低/中/高/严重）**: 高
