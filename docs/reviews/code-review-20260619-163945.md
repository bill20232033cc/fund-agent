- **入口/函数**: `_select_current_stage_candidate_evidence` → `_extend_current_stage_section_records` → `_matches_guarded_current_stage_source`
- **文件(行号)**: `fund_disclosure_processor.py:1871-1883`（主循环）、`:2124-2151`（guard 判断）、`:82-105`（`stage_status` match group 定义）
- **输入场景**: section/paragraph/table 的 heading 文本为 `报告期内基金投资策略和运作分析`（该字符串是 `holding_strategy_change` 的 strong token，同时在 `stage_status` guard tokens 中作为 guard 上下文许可词）
- **实际分支**: role 遍历顺序为 `stage_status` → `manager_change` → `share_scale_change` → `holding_strategy_change`。当处理 `stage_status` 时：
  1. strong tokens（如 `当前阶段` `所处阶段` `基金运作情况` `报告期内基金运作`）均未命中（`报告期内基金投资策略和运作分析` 不含 `报告期内基金运作` 作为连续子串）
  2. generic token `运作` 命中
  3. guard token `报告期内基金投资策略和运作分析` 命中 guard_context（section 级 guard_context ≡ texts）
  4. → `stage_status` 认领该 source path，加入 `seen_paths`
- **预期行为**: 该文本是 `holding_strategy_change` 的 strong token，语义上描述投资策略与运作分析而非"当前阶段"状态，应由 `holding_strategy_change` 认领
- **实际行为**: `holding_strategy_change` 遍历到同一 source path 时被 `seen_paths` 去重跳过，该节被错误标注为 `role=stage_status`
- **直接证据**:
  - `stage_status` guard tokens 定义（`:96-104`）包含 `"报告期内基金投资策略和运作分析"`
