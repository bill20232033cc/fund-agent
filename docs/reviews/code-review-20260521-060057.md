## Review Notes

- `LensApplicationPlan` 和 `LensChapterApplication` 保持在 Fund Capability template 层，没有把基金类型和 lens 应用逻辑上移到 Service/UI/Engine。
- `build_lens_application_plan(...)` 对未知基金类型、空章节、重复章节和越界章节均 fail closed，且通过 `resolve_preferred_lens(...)` 保留 CHAPTER_CONTRACT 真源关系。
- Renderer 只在第 0 章 `当前最值得盯住的变量` 和第 1 章 `看这类基金最先看什么` 两个确定性 slot 应用 normalized lens labels，没有把 raw `lens:` statements 粘贴进报告 Markdown。
- `TemplateRenderResult.lens_application_plan` 让应用结果可测试、可审计；`ProgrammaticAuditInput` 形状未改变。
- 缺失路径区分清楚：完全缺失 `basic_identity.value` 时保留显式数据不足输出并返回 `lens_application_plan=None`；存在 identity 但缺少或给出非法 `classified_fund_type` 时抛出 `ValueError`。
- Review 期间补强了 `index_fund` renderer 测试，额外断言 lens 应用后的 `ProgrammaticAuditInput` 仍可通过 programmatic audit。
