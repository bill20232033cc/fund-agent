## Findings

### F1-未修复-[高]-Eastmoney 来源包装器没有把 schema/integrity 失败稳定映射进 fallback taxonomy

- **入口/函数**: `FundDocumentRepository.load_annual_report()` → `AnnualReportPdfAdapter.fetch_pdf()` → `AnnualReportSourceOrchestrator.fetch_annual_report_pdf()` → `EastmoneyAnnualReportSource.fetch_annual_report_pdf()`
- **文件(行号)**: `AGENTS.md:77-79`、`AGENTS.md:232-243`、`fund_agent/fund/documents/sources.py:40-42`、`fund_agent/fund/documents/sources.py:522-553`、`fund_agent/fund/documents/sources.py:630-653`、`fund_agent/fund/pdf/downloader.py:39-45`、`fund_agent/fund/pdf/downloader.py:96-124`
- **输入场景**: EID 主源返回 `not_found` / `unavailable` 后进入 Eastmoney fallback；此时 akshare 返回缺少 `公告标题` / `报告ID` 的 DataFrame，或 Eastmoney PDF URL 返回 HTML / 非 PDF 内容
- **预期行为**: 年报来源必须按 `not_found`、`unavailable`、`schema_drift`、`identity_mismatch`、`integrity_error` 显式分类；只有 `not_found` / `unavailable` 可 fallback，`schema_drift` / `identity_mismatch` / `integrity_error` 必须 fail-closed，并保留 source、category、message 和原始 cause
- **实际行为**: `_download_pdf()` 对非 PDF 文件头抛 `ValueError`，但 Eastmoney wrapper 把 `ValueError` 统一转成 `AnnualReportSourceUnavailableError`，进入 fallback-eligible 类别；`_find_annual_report_id()` 直接访问 `df["公告标题"]` 和 `row["报告ID"]`，schema 漂移会以原始 `KeyError` 等异常绕过 orchestrator 的结构化分类
- **直接证据**: downloader 对非 PDF 响应的测试只断言 `ValueError` 和“不写缓存”（`tests/fund/pdf/test_downloader.py:236-284`）；Eastmoney wrapper 测试只覆盖 HTTP/OSError → unavailable（`tests/fund/documents/test_annual_report_sources.py:1284-1365`）；orchestrator 已有 fail-closed 测试只在 fake source 显式抛 `AnnualReportSourceIntegrityError` / `AnnualReportSourceSchemaError` 时成立（`tests/fund/documents/test_annual_report_sources.py:1041-1145`）
- **影响**: 当前默认来源链只有 EID→Eastmoney 时，非 PDF 可能最终被报告成 `unavailable` 或聚合失败，而不是 `integrity_error`；未来如果 Eastmoney 后再加第三来源，损坏 PDF / HTML 响应会被当作临时不可用继续 fallback，静默掩盖来源完整性问题。schema 漂移则会丢失统一 failure record，破坏 source provenance 和后续 evidence gate 的可审计性。
- **建议改法和验证点**: 在 Eastmoney adapter 边界做 typed translation：akshare 缺列/字段形态错误 → `AnnualReportSourceSchemaError`；PDF Content-Type 或 `%PDF-` 校验失败 → `AnnualReportSourceIntegrityError`；HTTP/timeout/缓存写入错误才是 `AnnualReportSourceUnavailableError`。补测试：Eastmoney 非 PDF 响应映射 `integrity_error` 并阻断第三来源；akshare 缺 `公告标题` / `报告ID` 映射 `schema_drift` 并保留 prior eligible failures；现有 HTTP/OSError unavailable 行为不回归。
- **修复风险（低/中/高）**: 中。会改变错误语义，可能影响依赖旧 `unavailable` 行为的脚本或历史 evidence 说明，但这是对硬约束的收口。
- **严重程度（低/中/高/严重）**: 高

### F3-未修复-[低]-来源元数据反序列化会把字符串 `"false"` 误读为 `fallback_used=True`

- **入口/函数**: `AnnualReportDocumentCache` / `ParsedAnnualReport.from_dict()` 读取缓存元数据，再由 `FundDataExtractor.extract()` 投影为公共 source provenance
- **文件(行号)**: `fund_agent/fund/documents/models.py:96-128`、`fund_agent/fund/documents/models.py:171-190`、`fund_agent/fund/source_provenance.py:126-176`、`docs/implementation-control.md:97`
- **输入场景**: 缓存、迁移脚本、人工修复或外部工具写入 JSON：`{"source":"eastmoney","fallback_used":"false"}`；该值在 JSON 层是字符串，不是布尔值
- **预期行为**: `fallback_used` 是 source provenance 安全字段，应只接受真实 bool，或显式解析有限字符串；非法类型不能依赖 Python truthiness 静默改义
- **实际行为**: `AnnualReportSourceMetadata.from_dict()` 使用 `bool(payload.get("fallback_used", False))`，Python 中 `bool("false")` 为 `True`；公共投影随后进入 fallback-used 分支，输出 `unknown_public_metadata_absent`
- **直接证据**: 当前 round-trip 测试覆盖正常 bool 和 legacy/unknown category，但未覆盖字符串 `"false"`（`tests/fund/documents/test_cache.py:134-183`）；控制文档已把该问题登记为 `Source metadata strict bool parsing` residual（`docs/implementation-control.md:97`）
- **影响**: 当前生产写入路径用 `to_dict()` 写真实 bool，因此不是 happy path 高危；但 evidence gate 或人工缓存恢复一旦混入字符串布尔，会把“主源成功无 fallback”污染成“使用 fallback 但缺上游失败类别”，误导来源覆盖判断。
- **建议改法和验证点**: 增加 strict bool parser：仅接受 bool，或显式映射 `"true"` / `"false"`；其它类型抛 `ValueError` 并沿用现有缓存降级策略。补测试覆盖 `"false"`、`"true"`、`0/1`、非法字符串；同时审视 `AnnualReportCacheProvenance.from_dict()` 的 bool 字段。
- **修复风险（低/中/高）**: 中，需要确认历史缓存兼容策略。
- **严重程度（低/中/高/严重）**: 低
