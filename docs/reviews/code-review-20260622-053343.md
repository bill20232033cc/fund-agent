### 001-未修复-中-`_proof_failure_reason` 未覆盖 `_reference_matches_anchor` 中多数匹配失败原因

- **入口/函数**: `_proof_failure_reason` → `_dimension_proof_boundary` → `_confirm_fact_v2`
- **文件(行号)**: `fund_agent/fund/evidence_confirm.py:939-970`
- **输入场景**: reference 的 `section_id`、`page_number`、`table_id`、`row_locator` 或 `anchor.document_year` 与当前章节 anchor 不匹配（但 `source_kind` 相同且 `report_year` 也匹配或未设置）。
- **实际分支**: `_reference_matches_anchor`（行 1489-1530）对上述字段逐一比较后返回 `False` → `_reference_is_proof` 返回 `False` → `_dimension_proof_boundary` 行 908 调用 `_proof_failure_reason`。但 `_proof_failure_reason` 中各条件检查未命中任何具体分支，落入行 970 的 fallback 返回通用消息 `"reference 不满足 proof predicate。"`。
- **预期行为**: 对 `anchor.document_year` 不匹配、`section_id` 不匹配、`page_number` 不匹配、`table_id` 不匹配、`row_locator` 不匹配分别给出具体中文失败原因。
- **实际行为**: 上述所有情况均返回相同的通用消息，消费者无法从 issue 消息中区分到底是哪个锚点身份字段不匹配。
- **直接证据**:
  - `_proof_failure_reason` 行 958–968 覆盖了 `candidate_only`、`not_proven`、`source_kind` 闭集外、`kind/source_kind` 组合不匹配、`anchor.source_kind` 不一致、`document_year` vs `report_year` 的六种情况，但未覆盖 `_reference_matches_anchor` 行 1512–1529 的另外五种字段级不匹配。
  - 行 970 `return "reference 不满足 proof predicate。"` 是唯一的 fallback。
- **影响**: proof_boundary 维度产生 E3 blocking issue 时，issue message 丢失了具体不匹配字段信息，降低了下游排查效率。不影响 gate 判定结果（都正确 fail），影响的是可操作性。
- **建议改法和验证点**: 在 `_proof_failure_reason` 中为 `_reference_matches_anchor` 的每种不匹配原因添加独立消息分支，或在 `_dimension_proof_boundary` 中直接构造更具体的 issue message 而不完全依赖 `_proof_failure_reason`。验证：对每种字段级不匹配各写一个断言 issue message 包含对应字段名的测试。
- **修复风险（低）**: 只改消息文本，不改变分支逻辑和 gate 语义。
