- The built-in manifest has no `programmatic_marker` rules, so the runtime branch is primarily future-proofing. The plan controls this by requiring fixture coverage if the branch is implemented in S1.
- Chapter 7 currently uses the required item text `危级/降级阈值`; the plan now matches that exact code text. If the template later corrects this wording, the coverage manifest validation should fail until the mapping is updated.
