# RR-09 Product Provenance Tier Contract Aggregate Deepreview

## Findings

### AGG-S2-001 - `reference_build_result=None` plus V2 pass bypasses provenance floor

- Severity: blocking
- Entry point: `summary_from_repository_result()` output consumed by `_evidence_confirm_quality_gate_issues()`.
- Triggering input: repository runner returns `pathway_status="pass"`, deterministic V2 `overall_status="pass"`, and `reference_build_result=None`.
- Actual branch:
  - `fund_agent/fund/evidence_confirm_production.py:447` through `:452` returns `_not_run_provenance_contract()` when `reference_build_result is None`.
  - That produces `provenance_status="not_run"`, `minimum_provenance_tier="none"`, and `provenance_missing_fact_count=0`.
  - `fund_agent/fund/quality_gate_integration.py:232` only blocks when `summary.provenance_status == "fail"` or `summary.provenance_missing_fact_count` is non-zero.
  - Because deterministic status is `pass`, the fallback ECQ2 deterministic fail branch at `fund_agent/fund/quality_gate_integration.py:262` is also skipped.
- Expected behavior: a checked applicable fact without any repository-bounded reference should not pass the section-or-better provenance floor; it should be visible as claim provenance missing and therefore produce ECQ2/block.
- Actual behavior: the summary is not a provenance pass, but quality gate emits no ECQ issue, so a product run can continue with no claim provenance.
- Direct evidence:
  - Accepted controller judgment requires `section` as minimum release floor and states that `reference_build_result=None` plus V2 pass must not be treated as provenance pass.
  - Accepted plan states `provenance_status="pass"` only when every checked applicable fact has section-or-better proven provenance, and S2 must block ECQ2 provenance missing.
  - Current regression only asserts `provenance_status=="not_run"` for missing `reference_build_result` in `tests/fund/test_evidence_confirm_production.py:403`; there is no quality-gate integration test for this summary shape.
- Impact: a degraded or skewed repository runner result can bypass the product provenance contract while still showing deterministic V2 pass. This undermines the release-floor distinction S1/S2 were introduced to enforce.
- Fix direction: classify `reference_build_result=None` with existing checked applicable V2 facts as provenance missing, or make S2 treat `provenance_status="not_run"` plus checked facts as ECQ2/block. Add focused tests that prove missing reference build plus V2 pass maps to ECQ2/block.
- Fix risk: keep repository/pathway failure and true Evidence Confirm not-run/off summaries as `ECQ0` or `ECQ1` per existing policy; the fix should only affect pass-path V2 facts lacking reference provenance.
