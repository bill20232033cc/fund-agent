## Findings

### 001-未修复-[中]-section context resolution is recomputed inside every matcher scan
- **入口/函数**: `extract_docling_template_fields()` -> `_match_field()` -> matcher loops -> `_effective_section_id()`
- **文件(行号)**: `fund_agent/fund/documents/candidates/template_field_extraction.py:446-448`, `498-500`, `535-537`, `575-577`, `614-616`, `666-668`, `1074-1100`
- **输入场景**: A real Docling candidate envelope with hundreds of text blocks/tables and default 23 target fields.
- **实际分支**: Every matcher loop calls `_effective_section_id(document, block)`. `_effective_section_id()` calls `map_candidate_locator_to_anchor_candidate(...)`, which rebuilds candidate section resolution for the same block repeatedly across target fields.
- **预期行为**: Section context should be resolved at most once per block per extraction run, then reused by all field matchers.
- **实际行为**: The same table/text block can be remapped for every field group that scans it. The re-evidence command over four accepted envelopes took roughly 29 seconds even though it only reused existing local JSON artifacts.
- **直接证据**: The changed matchers call `_effective_section_id()` inside loops over `document.tables` / `document.text_blocks`; `_effective_section_id()` has no cache and always calls `map_candidate_locator_to_anchor_candidate(...)`.
- **影响**: Candidate no-live evidence and future remediation loops become materially slower; this can make iterative gates expensive and hide performance regressions before baseline readiness work.
- **建议改法和验证点**: Add extraction-scope cache keyed by block identity, pass it through matcher calls, and verify section-context tests plus re-evidence still produce coverage uplift.
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 中
