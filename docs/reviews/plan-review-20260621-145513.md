## Findings

### 001-未修复-中-auditability_score 类型与 not_applicable 语义冲突

- **位置**: `EvidenceConfirmFactResult` dataclass and `Auditability Score`
- **问题类型**: 契约缺失 / 不可直接实施
- **当前写法**: `EvidenceConfirmFactResult.auditability_score: int`，但 per fact scoring 又要求 `not_applicable`: derived/not-applicable fact。
- **反例/失败场景**: implementation agent 要么把 `not_applicable` 塞进 `int` 字段导致类型不一致，要么把 not-applicable fact 记成 `0`，使 aggregate score 把 derived/not-applicable facts 当作失败。
- **为什么有问题**: 当前 `ChapterFactStatus` 已有 `not_applicable`，Evidence Confirm 计划也明确 derived/not-applicable 不应强制走 annual-report excerpt matching。score contract 必须能表达“不计分”。
- **直接证据**: `fund_agent/fund/chapter_facts.py` 定义 `ChapterFactStatus = Literal["available", "missing", "not_applicable", "unavailable", "unknown"]`；plan 同时写 `auditability_score: int` 和 per fact `not_applicable`。
- **影响**: 生成错误代码 / aggregate score 失真 / review 不可验收。
- **建议改法和验证点**: 把 per-fact score 字段改成 `auditability_score: int | None`，并规定 `None` 只用于 `status="not_applicable"`；aggregate average 只统计非 `None` 分数，测试覆盖 derived/not-applicable 不拉低 aggregate score。
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 中

### 002-未修复-中-candidate route 判定写成 looks like 不可实施

- **位置**: `Candidate and proof boundary`
- **问题类型**: 契约缺失 / 测试缺口
- **当前写法**: `source_kind` looks like a candidate route 时不能满足 E2/E3。
- **反例/失败场景**: 一个 reference 写 `source_kind="docling_pdf_candidate"` 但 `candidate_only=False`、`source_truth_status="proven"`；另一个写 `source_kind="annual_report"` 但 `candidate_only=True`。如果实现自行解释 “looks like”，不同实现会接受不同候选证据，导致 candidate proof boundary 漂移。
- **为什么有问题**: 本仓库对 candidate-only / not_proven 边界是硬约束；计划必须给出 closed rule，而不是让实现者猜字符串模式。
- **直接证据**: memory/control facts and design docs repeatedly require candidate evidence remain `candidate_only / not_proven / NOT_READY`; plan itself把 candidate boundary列为成功测试，但没有列出 deterministic predicate。
- **影响**: implementation agent 可能错误 promotion candidate evidence / review 难以判断实现是否正确。
- **建议改法和验证点**: 明确 predicate：reference 可确认必须同时满足 `candidate_only is False`、`source_truth_status == "proven"`、`source_kind in {"annual_report", "reviewed_note", "derived"}`，且 `reference_kind` 与 `source_kind` 合法组合；其它全部作为 non-proof issue。测试覆盖 `docling_pdf_candidate`、`eid_html_render_candidate`、`pdfplumber_pdf_candidate`、`annual_report + candidate_only=True`。
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 中
