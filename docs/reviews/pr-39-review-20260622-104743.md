### 001-未修复-[中]-V1 仍会把部分悬空 anchor 的 fact 判为 pass，和 PR 的 dangling-anchor fail-closed 口径不一致

- **入口/函数**: `confirm_chapter_evidence()` -> `_confirm_fact()` -> `_proof_references_for_fact()`
- **文件(行号)**: `fund_agent/fund/evidence_confirm.py:1381`, `fund_agent/fund/evidence_confirm.py:1486`
- **输入场景**: 一个 `available` fact 同时声明两个 `evidence_anchor_ids`，其中一个存在于当前章节 `evidence_anchors` 且有 valid proven reference，另一个 anchor id 不存在于当前章节。
- **实际分支**: `_confirm_fact()` 只在 `not known_anchors` 时失败；只要至少一个 anchor 存在就继续执行。随后 `_proof_references_for_fact()` 遍历 `fact.evidence_anchor_ids` 时对缺失 anchor 执行 `continue`，只保留存在 anchor 的 proven reference。只要 value match 命中，V1 返回 `pass` / `100`。
- **预期行为**: PR body 声明本 PR 对 dangling anchors fail closed；phase-1 accepted plan 的 V1 score contract 也把 `dangling/no reference for required fact` 归入 `0` 分。部分悬空 anchor 不能被一个有效 anchor 掩盖。
- **实际行为**: 复现命令返回 `pass 100 pass 100 ()`，即整体状态、fact 状态和分数均为通过，且无 issue。
- **直接证据**:
  - `fund_agent/fund/evidence_confirm.py:1381-1384` 只处理“没有任何 known anchor”的情况，没有处理“部分 anchor 悬空”。
  - `fund_agent/fund/evidence_confirm.py:1486-1489` 对缺失 anchor 直接 `continue`，导致 dangling anchor 不进入 issue 或 score。
  - `docs/reviews/evidence-confirm-anchor-auditability-score-plan-20260621.md:194-198` 明确 V1 `0` 分包含 `dangling/no reference for required fact`。
  - `docs/reviews/evidence-confirm-scoring-v2-aggregate-fix-evidence-20260622.md:8-16` 说明当前 dangling-anchor fix 被限定为 V2，且 V1 public functions remain untouched。
  - PR body summary 未限定 dangling-anchor fail-closed 只适用于 V2。
- **影响**: V1 public helper 仍可对 anchor linkage 不完整的 required fact 产出 pass-like 结果；如果调用方继续使用 V1，PR body 和 phase-1 contract 会高估 Evidence Confirm 的 fail-closed 覆盖。影响局限于 no-live explicit-reference helper，但属于 public behavior / PR truthfulness mismatch。
- **建议改法和验证点**: 若 dangling-anchor fail-closed 是 PR-39 对所有 Evidence Confirm public helpers 的承诺，在 V1 `_confirm_fact()` 中复用 `_dangling_anchor_ids_for_fact()`，在发现任一 dangling id 时返回 E3 blocking、score `0`，并新增 V1 regression：valid anchor + dangling anchor + valid proven reference 时 `overall_status == "fail"`、fact score `0`、issue anchor 指向 dangling id。若 controller 决定 V1 必须保持该旧行为，则至少收窄 PR body 和 phase-1/V1 文档口径，明确 dangling-anchor fix is V2-only。
