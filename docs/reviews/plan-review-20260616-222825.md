## 3. Findings

### F1-未修复-高-repository_reference_rows 未定义导致 implementation worker 必须重新设计 source-reference 契约

- **位置**: Section 5.2 Public Helper API; Section 5.3 Required Data Model; Section 7 Evidence Slice.
- **问题类型**: 契约缺失 / 不可直接实施 / 测试缺口
- **当前写法**: The helper API requires `repository_reference_rows: Mapping[str, object]`, but the plan does not define the shape of a repository reference row, how it is derived from `ParsedAnnualReport`, how source snippets from `source_truth_matrix.json` relate to it, or how table row/column labels are represented.
- **反例/失败场景**: Implementation worker may invent one of several incompatible approaches: read `source_excerpt_samples` from the previous matrix as if they were authoritative source rows; call `FundDocumentRepository` inside the helper; use direct parsed-cache/PDF paths; or build row labels ad hoc from table text. Each option would make the evidence hard to review and could violate the repository/no-direct-cache boundary.
- **为什么有问题**: The accepted `source -> processed -> fund` contract requires source-layer proof to be distinct from processed-layer locator and fund semantic proof. Without a concrete repository reference row schema, the source layer is still implicit. The plan therefore does not yet remove the one-off script risk it is meant to solve.
- **直接证据**:
  - Plan Section 5.2 lists `repository_reference_rows: Mapping[str, object]` but no fields.
  - Plan Section 5.6 requires `source_layer_status` values but does not state what source-reference evidence can produce them.
  - Controller judgment requires using accepted same-source evidence and repository-mediated references, not direct PDF/cache/source-helper body access.
- **影响**: Implementation may be non-reviewable, overfit to previous excerpts, or cross the repository boundary. Evidence could appear to close residuals without an auditable same-source row contract.
- **建议改法和验证点**:
  - Add a `RepositoryReferenceRow` / `RepositoryReferenceTableCell` schema to the plan.
  - Define the evidence wrapper, not the pure helper, as the only code allowed to call `FundDocumentRepository.load_annual_report(..., force_refresh=False)` under no-live/cache-only guard semantics.
  - Require each reference row/cell to include `fund_code`, `document_year`, `repository_source_name`, `source_mode`, `section_id`, `page_number`, `table_id`, `row_index`, `column_index`, `row_label_path`, `column_header_path`, `normalized_text`, `raw_text`, `source_body_match_count_scope`, and `reference_origin`.
  - Add tests proving the pure helper does not call repository or read files, and that missing reference rows produce `blocked_reference_unavailable`.
- **修复风险（低/中/高）**: 中
- **严重程度（低/中/高/严重）**: 高
