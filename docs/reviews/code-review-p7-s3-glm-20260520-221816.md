逐项审查结论：

1. **EID primary → Eastmoney fallback 顺序**: `AnnualReportSourceOrchestrator(None)` 在 `sources.py:558-561` 构造 `(EidAnnualReportSource(config=self.config), EastmoneyAnnualReportSource())`，EID 在前。`AnnualReportSourceOrchestrator(())` 在 `sources.py:563-564` 抛出 `ValueError`。`test_orchestrator_rejects_empty_sources_but_none_uses_default` 验证两个分支。

2. **Fail-closed mismatch/schema vs fallback-eligible not-found/unavailable**: `sources.py:599-606`，orchestrator 对 `NotFoundError` 和 `UnavailableError` 执行 `continue`（fallback-eligible），对 `MismatchError` 和 `SchemaError` 直接 `raise`（stops fallback）。`_select_eid_annual_report_candidate`（`sources.py:930-972`）对空候选返回 `NotFoundError`，对字段矛盾返回 `MismatchError`，对重复有效候选和附件链路返回 `SchemaError`。测试覆盖：not-found fallback（`test_orchestrator_falls_back_to_eastmoney_after_eid_not_found`），mismatch 不 fallback（`test_orchestrator_does_not_fallback_after_eid_mismatch`）。

3. **validate_fund/search/PDF endpoint 解析**: `_parse_eid_validate_response`（`sources.py:793-836`）处理 `isSuccess` true/false/null 三种情况。`_parse_eid_search_response` + `_parse_eid_candidate`（`sources.py:868-927`）类型化解析 `aaData` 行。`_validate_pdf_response`（`sources.py:1030-1049`）验证 `Content-Type: application/pdf` 和 `%PDF-` magic bytes。`_build_eid_ao_data`（`sources.py:839-865`）编码 `reportType=FB010`、`reportYear`、`fundCode`。

4. **Request-level timeout 语义**: `_request_with_retries`（`sources.py:739-790`）接受 `timeout` 参数，每次调用传给 `client.post(..., timeout=timeout)` 或 `client.get(..., timeout=timeout)`。`_validate_fund` 和 `_search_annual_report` 传 `metadata_timeout_seconds`（`sources.py:413, 437`），PDF 请求传 `pdf_timeout_seconds`（`sources.py:374`）。单一 `async with self._client_factory() as client` 会话（`sources.py:357`）保证 cookie 保持。`test_eid_source_uses_distinct_request_level_timeouts` 用 `_RecordingEidClient` 断言三个 timeout 值为 `(3.5, 3.5, 77.0)`。

5. **确定性缓存路径**: `_build_pdf_cache_path`（`sources.py:450-464`）返回 `{cache_dir}/{fund_code}_{year}_annual_report_eid.pdf`，与基金代码和年份确定性绑定。

6. **force_refresh 行为**: `sources.py:368-369`，缓存文件存在且 `force_refresh=False` 时直接返回带 metadata 的结果，不调用 PDF endpoint。`force_refresh=True` 跳过缓存检查，重新下载并写入。`test_eid_source_force_refresh_overwrites_cached_pdf` 和 `test_eid_source_without_force_refresh_reuses_existing_pdf_after_metadata_validation` 分别验证。

7. **无 Service/UI/Engine/CLI/cache schema/extra_payload 泄漏**: grep `fund_agent/services/`、`fund_agent/ui/`、`fund_agent/engine/`、`fund_agent/cli.py` 均无 `documents.sources`、`EidAnnualReportSource`、`AnnualReportSourceOrchestrator` 的 import。EID 实现完全封装在 `documents` 模块内。

8. **测试与 plan 合规**: plan 要求 18 项测试，实际文件覆盖全部 18 项（tests 5-8 合并为一个 parameterized test `test_eid_source_rejects_mismatched_candidates`）。所有 EID 测试使用 `httpx.MockTransport` 或 `_RecordingEidClient`，不依赖 live network。附加测试覆盖 attachment candidate rejection（`test_eid_source_rejects_attachment_candidate`），超出 plan 要求。

## Residual Risk

- EID 响应 schema 可能随时间漂移；当前 fail-closed 设计（`_parse_eid_candidate`、`_select_eid_annual_report_candidate`）会拒绝不符合预期的响应，但缺少 schema drift 的主动监测或告警。
- `_request_with_retries` 对 5xx 和非 200 响应统一视为 unavailable 并重试，但未区分 4xx 客户端错误中是否有可恢复的情况（当前行为对所有非 200/5xx 直接 unavailable，安全但可能过度保守）。
- 长期运行中 EID cookie/session 行为依赖 httpx 默认实现，若 EID 站点变更 session 策略，单一 AsyncClient 会话可能不足。
- 测试未覆盖 EID 响应字段包含非预期类型时的 `_required_string` / `_optional_string` / `_optional_int` 防御路径（当前通过类型化 `_EidAnnualReportCandidate` 间接保护，风险低）。
