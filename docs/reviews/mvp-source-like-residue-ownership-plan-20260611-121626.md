## Recommended Disposition

Main recommendation: `delete`

Rationale:

- `promote-none` keeps source-like residue in the product namespace and does not clear release/readiness hygiene.
- `archive` turns a local workstation helper into repository evidence-chain material; this helper is not product evidence or design input.
- `move-to-local-tooling` would retain a non-product tool that writes user home config and token fields, and would require a new tooling directory, ignore rule and maintenance owner. That is outside this gate.
- `delete` is the smallest disposition that removes non-product, secret-adjacent local helper code from `fund_agent/`.

Important: deletion is destructive. It must not happen in this planning gate. It requires a reviewed implementation gate and explicit authorization.
