### Implementation Compliance

1. **Data Structures** (lines 66-78, 402-412):
   - `_CORE_RISK_REQUIRED_TOP_LEVEL`: 5 required keys defined correctly
   - `_CORE_RISK_ROLE_KEYS`: 4 role keys defined correctly
   - `_CoreRiskRoleValueCandidate`: frozen dataclass with `output_path`, `role`, `value`, `anchor`, `source_field_path`
   - `_CoreRiskValueCandidate`: union type merging `_RiskCharacteristicValueCandidate | _CoreRiskRoleValueCandidate`

2. **Core Extraction Logic** (lines 4689-5061):
   - `_select_core_risk_role_values()`: correctly iterates `_CORE_RISK_MATCH_GROUPS` skipping `risk_characteristic`, collects paragraph and cell candidates, resolves conflicts
   - `_collect_core_risk_role_paragraph_candidates()`: enforces `locator_stability == "stable"`, prioritizes `text_normalized` over `text_raw`, applies guard context check, rejects heading-only
   - `_collect_core_risk_role_cell_candidates()`: enforces stable locator, excludes header cells/rows, rejects placeholders, applies guard context
   - `_is_core_risk_role_heading_only()`: correctly rejects text that equals tokens, or short text (< 15 chars) without ASCII digits
   - `_resolve_core_risk_role_candidate()`: returns first candidate when all normalized texts are identical, adds to `ambiguous_paths` on conflict
   - `_build_core_risk_value()`: dispatches by type (`_RiskCharacteristicValueCandidate` vs `_CoreRiskRoleValueCandidate`), returns empty dict if no values

3. **Gap/Status Semantics** (lines 5092-5170):
   - `_core_risk_source_truth_gaps()`: emits `ambiguous_table_or_locator` for conflicts, `field_family_missing` when empty, `field_family_partial` for missing required keys
   - `_core_risk_status()`: returns `"accepted"` when all 5 required keys present, `"partial"` when some present, `"missing"` when none
   - No `deferred_role` gaps emitted (verified by grep)

4. **Token Matching** (lines 698-767, 8744-8771):
   - `_CORE_RISK_MATCH_GROUPS`: defines strong tokens, generic tokens, and guard tokens for each role
   - `_matches_guarded_core_risk_source()`: strong token match → True; generic token + guard context → True; otherwise False
   - Guard context includes heading_path, table headings, row/column labels

5. **Facade No-Projection** (verified by test `test_explicit_disclosure_core_risk_role_keys_not_projected_to_bundle`):
   - No `StructuredFundDataBundle.core_risk` added
   - Role keys not projected to bundle fields
   - Only `risk_characteristic_text` fallback preserved

6. **Anchor Handling** (lines 5214-5234):
   - `_dedupe_anchors()`: deduplicates by dataclass value, preserves order
   - Role anchors use correct `row_locator` format with field key and source path
   - No `source_anchors` embedded in role subvalues
