## Findings

### P7S2-PR1-未修复-中-所有 fallback-eligible 失败被压扁成 FileNotFoundError 会误报源不可用
- **位置**: `Fallback Error Categories` / `Source Orchestrator`
- **问题类型**: 契约缺失 / 状态机漏洞 / 可观测性缺口
- **当前写法**: Plan 要求 `AnnualReportSourceUnavailableError` 和 `AnnualReportSourceNotFoundError` 都允许 fallback；但在所有 sources 都失败时，统一 “raise `FileNotFoundError` with a concise message that includes source names and categories”。
- **反例/失败场景**: P7-S3 后 EID timeout，Eastmoney/akshare 也 5xx。按当前 plan，最终会抛 `FileNotFoundError`，上层和质量日志会把“外部源暂不可用”误读为“该基金该年份无年报”。
- **为什么有问题**: P7-S1 已要求 fallback 区分 unavailable/not-found 与 contradictory official metadata。P7-S2 是错误分类边界，如果这里把 unavailable 压扁成 not-found，后续 implementation 很难恢复真实 root cause。
- **直接证据**: P7-S2 plan 写明 fallback-eligible errors 包含 network timeout/HTTP 5xx/rate limit，也写明 all sources fallback-eligible failures 最终抛 `FileNotFoundError`。
- **影响**: 抽取失败原因失真、source reliability 统计不可审计、真实网络故障被缓存/summary/用户提示误判为缺文档。
- **建议改法和验证点**: 修改 plan：只有所有失败均为 `AnnualReportSourceNotFoundError` 时最终抛 `FileNotFoundError`；只要存在 `AnnualReportSourceUnavailableError`，最终应抛 `AnnualReportSourceUnavailableError` 或显式 aggregate source error，并在 message 中保留每个 source 的 category。新增 fake test：primary unavailable + fallback unavailable 最终不是 `FileNotFoundError`；primary not-found + fallback not-found 才是 `FileNotFoundError`。
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 中
