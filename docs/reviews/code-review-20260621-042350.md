# Code Review: Extractor Output Repository

## Findings

### 001-未修复-高-Service 保存前未校验 extractor 返回 bundle 身份

- **Entry point**: `ExtractorOutputService.save(...)`
- **Triggering input**: `ExtractorOutputSaveRequest(fund_code="004393", report_year=2024, report_type="annual_report", ...)`，但 injected 或 future extractor 返回 `StructuredFundDataBundle(fund_code="110011", report_year=2024, ...)`
- **Actual branch**: Service 校验 request 后直接调用 `repository.save(bundle=bundle, report_type=request.report_type)`
- **Expected behavior**: Service 应在保存前验证 `bundle.fund_code == request.fund_code` 且 `bundle.report_year == request.report_year`，不一致时 fail-closed
- **Actual behavior**: Repository 以 bundle 自身身份决定路径，可能把请求 `004393` 的运行结果保存成 `110011/annual_report/2024/structured_fund_data.json`，CLI 仍返回成功
- **Direct evidence**: `fund_agent/services/extractor_output_service.py` 的 `save()` 只调用 `_validate_request(request)`，随后把 extractor 返回的 bundle 原样传给 repository；`ExtractorOutputRepository.save()` 使用 `bundle.fund_code` / `bundle.report_year` 构造 identity
- **Impact**: 用户请求身份、写盘路径和返回身份可能不一致；多年分析或 LLM route 后续按用户请求寻找 artifact 时缺失，或消费错误基金输出；该错误不会被当前 Service/CLI 测试发现
- **Fix direction**: 在 Service 层新增 `_validate_bundle_identity(request, bundle)`，保存前校验 fund/year；新增 Service 测试覆盖 mismatched bundle 不调用 repository
- **Fix risk**: 低
- **Severity**: 高

### 002-未修复-高-Repository load 未校验 bundle 内部身份与顶层身份一致

- **Entry point**: `ExtractorOutputRepository.load(...)`
- **Triggering input**: 路径和顶层 JSON identity 是 `110011/annual_report/2024`，但 `payload["bundle"]["fund_code"]` 或 `payload["bundle"]["report_year"]` 被手工或旧版本工具写成其它值
- **Actual branch**: `load()` 校验顶层 `schema_version/fund_code/report_type/report_year` 后，只检查 `bundle` 是 dict，并直接返回 `bundle_payload`
- **Expected behavior**: load 应 fail-closed 校验 `bundle.fund_code` 和 `bundle.report_year` 与顶层 identity 一致
- **Actual behavior**: `ExtractorOutputRecord.identity` 可以显示 `110011/2024`，但 `record.bundle_payload` 内部携带 `999999/2024` 或 `110011/2025`
- **Direct evidence**: `fund_agent/fund/extractor_output_repository.py` 的 `load()` 在 `bundle_payload = payload.get("bundle")` 后没有读取或比较 `bundle_payload["fund_code"]` / `bundle_payload["report_year"]`
- **Impact**: 后续 consumer 读取 record 时会面对两个互相矛盾的身份真源；LLM route 或多年分析可能使用顶层 identity 做索引、用 bundle 内部 identity 做事实，从而污染跨年/跨基金输入
- **Fix direction**: 新增 `_require_bundle_identity_matches(bundle_payload, identity)`，校验 `fund_code` 和 `report_year`；新增 repository 测试分别覆盖 bundle fund_code/year mismatch
- **Fix risk**: 低
- **Severity**: 高
