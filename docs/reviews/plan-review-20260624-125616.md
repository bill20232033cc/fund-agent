## Findings

### PR-001-未修复-高-subfield scoped anchors can be fabricated from field-level anchors

- **位置**: A6-S1, exact changes 1-4; tests 1-2; risks table.
- **问题类型**: 契约缺失 / 测试缺口 / 架构边界
- **当前写法**: The plan says `_public_value_paths(...)` should return leaf paths under `output_field_name` for dict values, and `_build_field_family_result(...)` should replace `anchors.extend(field.anchors)` with field-scoped anchors. This permits cloning every existing field anchor into every leaf path of a composite value.
- **反例/失败场景**: A parsed annual `fee_schedule` field can contain `management_fee` and `custody_fee`, while its `field.anchors` only prove a field-level table or a legacy semantic locator, not which anchor corresponds to which subvalue. If implementation clones all anchors to both `fee_schedule.management_fee` and `fee_schedule.custody_fee`, the materializer can test custody-fee tokens against a management-fee anchor's table/section or vice versa. If the token appears uniquely in that table, Evidence Confirm could produce row-level proof for the wrong anchor-to-subvalue relationship.
- **为什么有问题**: A6 exists because A5 live evidence proved the runtime path lacks precise `field=` locators. The plan must not manufacture the missing subfield provenance from composite values alone. `active_annual.py` currently aggregates old extractor anchors with `anchors.extend(field.anchors)` and does not preserve an anchor-to-leaf mapping. The plan's proposed `_public_value_paths(...)` infers subfield scope from value shape rather than source evidence.
- **直接证据**:
  - `fund_agent/fund/processors/active_annual.py` `_build_field_family_result(...)` sets `value[mapping.output_field_name] = field.value` and then `anchors.extend(field.anchors)`.
  - Existing old extractors commonly build anchors with `row_locator=matched_field.field_name` or other legacy strings, not subfield paths.
  - A5 live evidence recorded `recognized_processor_row_locator_anchors=0` and `processor_row_locator_rows=0`, so there is no runtime proof that parsed annual anchors carry subfield identity.
- **影响**: Implementation agent could produce false row-level Evidence Confirm references, making V2 appear stronger while violating evidence traceability. This would be worse than the current coarse-reference failure because it converts insufficient proof into apparently precise proof.
- **建议改法和验证点**:
  - Split locator identity into two levels:
    1. safe top-level field identity can be attached when mapping ownership is known;
    2. subfield identity may only be attached when an extractor/FDD source already exposes direct anchor-to-subfield provenance.
  - For A6 no-live implementation, use `source_field_path=<top_level_field>` for parsed annual anchors unless direct subfield provenance exists.
  - Add a negative test proving a composite field with two subvalues and one field-level anchor does not create subfield-scoped row proofs.
  - Route actual subfield provenance to a later extractor-specific gate if needed.
- **修复风险（低/中/高）**: 中
- **严重程度（低/中/高/严重）**: 高
