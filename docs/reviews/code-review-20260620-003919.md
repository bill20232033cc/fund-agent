### 1. proof-positive source-truth admission is fail-closed

**Verdict: PASS**

`_validate_source_truth_admission()` (line 761-807) performs exhaustive fail-closed checks:
1. `content_intermediate` must exist (non-content FDD → `source_truth_admission_missing`)
2. `source_truth_admission` proof must be present (None → `source_truth_admission_missing`)
3. Proof must be `FundDisclosureSourceTruthAdmissionProof` instance (wrong type → `source_truth_admission_invalid`)
4. `candidate_boundary` must be None (non-None → `source_truth_admission_invalid`)
5. `failure_class` must be None (non-None → `source_truth_admission_invalid`)
6. `source_provenance` must not be None (None → `source_truth_admission_invalid`)
7. All identity fields must match dispatch key and intermediate (any mismatch → `source_truth_admission_invalid`)

The proof contract (`FundDisclosureSourceTruthAdmissionProof.__post_init__`, contracts.py line 226-261) enforces at construction time that all four boolean verifiers are `True`, proof_kind/source_boundary/document_kind/intermediate_kind/source_kind/producer are exact literals, and fund_code/report_year are valid.

Test coverage: `test_source_truth_admission_requires_positive_proof`, `test_source_truth_admission_marks_non_content_intermediate_missing`, `test_source_truth_admission_rejects_identity_mismatch`, `test_source_truth_admission_accepts_repository_loaded_identity_proof`.

### 2. candidate_boundary is None is necessary but not sufficient

**Verdict: PASS**

`_validate_source_truth_admission()` checks `content_intermediate.candidate_boundary is not None` at line 788-789 and returns `source_truth_admission_invalid` if present. This means `candidate_boundary=None` is necessary for source-truth admission.

However, `candidate_boundary=None` alone does NOT bypass the proof requirement: even when `candidate_boundary=None`, the proof must still exist, be of correct type, pass identity checks, and all other admission conditions must hold. This is confirmed by `test_product_essence_source_truth_requires_proof_even_when_candidate_boundary_none` (line 670-697): content with `candidate_boundary=None` but no proof produces `value={}`, `anchors=()`, and `source_truth_admission_missing` gap.

### 3. Missing/invalid proof never produces public values or anchors

**Verdict: PASS**

When `source_truth_gap_code` is not None (proof missing or invalid), `_field_families_for_intermediate()` (line 753-758) applies `_with_source_truth_admission_gap()` to ALL field families. The `product_essence_source_truth` extraction (line 715-718) is only invoked when `source_truth_extraction_allowed` is True (admission not blocked AND source_truth_gap_code is None).

For the five non-product-essence families: they always produce either `candidate_evidence` (with `status="missing"`, `value={}`, `anchors=()`) or pure `missing` status. No public value or anchor is ever emitted for these families regardless of proof state.
