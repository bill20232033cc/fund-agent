## Findings

### [P1] `fund_code` is validated after `strip()` but consumed unnormalized by quality gate and extraction

- Entry point: `fund-analysis analyze/checklist FUND_CODE`, and direct `FundAnalysisService.analyze()` / `checklist()` calls.
- Triggering input: a fund code with leading or trailing whitespace, e.g. `FundAnalysisRequest(fund_code="004393 ")`.
- Actual branch: `_validate_request()` strips the code into `normalized_fund_code` and accepts it because the stripped value is six digits, but `_run_analysis_core()` then calls `_check_pool_membership_before_extraction(request, ...)` and `self._extractor.extract(request.fund_code, ...)` with the original, unstripped `request.fund_code`.
- Expected behavior: once a fund code is accepted by validation, every downstream consumer in the same request should use the same canonical six-digit code.
- Actual behavior: the default product `block` quality gate checks `docs/code_20260519.csv` with the raw code. `004393 ` is treated as absent from the selected-fund CSV and raises `QualityGateNotRunBlockedError` before extraction, even though `004393` is present and is the README quick-start fund.
- Direct evidence:
  - `fund_agent/services/fund_analysis_service.py:776` creates `normalized_fund_code = request.fund_code.strip()` and validates that value at `fund_agent/services/fund_analysis_service.py:777-780`.
  - `_run_analysis_core()` immediately uses the original request value for membership and extraction at `fund_agent/services/fund_analysis_service.py:593-595`.
  - `_check_pool_membership_before_extraction()` passes the original request value to `check_quality_gate_fund_membership()` at `fund_agent/services/fund_analysis_service.py:822-825`.
  - Reproduction run:

```bash
uv run python - <<'PY'
import asyncio
from fund_agent.services.fund_analysis_service import FundAnalysisService, FundAnalysisRequest, QualityGateNotRunBlockedError

class NeverExtractor:
    def __init__(self): self.calls=[]
    async def extract(self, fund_code, report_year, *, force_refresh=False):
        self.calls.append((fund_code, report_year, force_refresh))
        raise AssertionError('should not extract')

async def main():
    ex = NeverExtractor()
    try:
        await FundAnalysisService(extractor=ex).analyze(FundAnalysisRequest(fund_code='004393 ', report_year=2024))
    except QualityGateNotRunBlockedError as e:
        print(type(e).__name__)
        print(e.reason)
        print('extractor_calls', ex.calls)

asyncio.run(main())
PY

```

- Impact: a visually valid user input can be rejected as a quality-gate not-run condition instead of being normalized. This is user-visible, affects both `analyze` and `checklist` in product mode, and can mask the true state of a selected fund as a data-quality failure.
- Fix direction: make canonicalization part of the Service contract, not only a local validation check. For example, return the normalized code from request validation or resolve a normalized request/contract object, then use that value consistently for quality-gate membership, extraction, default run id inputs, and any future Host/Agent request handoff. Add regression tests for `"004393 "` and `" 004393"` covering product `block` membership and extractor call arguments.
- Fix risk: low to medium. Normalizing accepted whitespace is likely intended, but the fix must ensure error messages and fake extractor tests assert the canonical code consistently and must not silently accept non-ASCII digits.
- Severity: P1.
