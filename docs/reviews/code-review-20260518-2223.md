## Review Notes

- CLI remains UI-only: it parses explicit parameters, validates enum-like CLI values, constructs `FundAnalysisRequest`, calls `FundAnalysisService`, and writes Markdown to stdout.
- Service orchestration respects boundaries: fund document access remains behind `FundDataExtractor`; Service coordinates P1 structured data, P2 analysis, template rendering, and programmatic audit without reading PDFs or cache files directly.
- Explicit parameters are not hidden in `extra_payload`; `FundAnalysisRequest` declares all Service inputs directly.
- Fund type priority is preserved: P1 profile extraction calls `classify_fund_type(report)` before building the generic profile fields, and Service refuses analysis when `classified_fund_type` is absent.
- The parsed report cache quality gate rejects low-quality cached reports before they can mask real extraction failures.
- The P3 CLI matrix exercises real Typer parsing, Service orchestration, Capability analysis, template rendering, programmatic audit, and evidence rendering while isolating network/PDF side effects through fake repository and NAV provider.
- P3-S4/P3-S5 close previous false-positive gaps by asserting `audit_result.checked_rules == ("P1", "P2", "P3", "L1", "R1", "R2")`, empty issues, 8 body evidence lines, appendix source anchors, and no missing-anchor placeholders.
- P3-S7/P3-S8 add reproducible gates for coverage and post-extraction Service latency.
- README now describes current user-facing behavior and does not present placeholder `checklist` or thermometer report wiring as completed.

## Residual Risks

- Real PDF/network smoke is still separate. P3 deterministic gates intentionally use fake repository and fake NAV provider for stability.
- Thermometer adapter is implemented and tested but not yet wired into CLI/Service report generation.
- The classifier could still benefit from a future cleanup that carries fund short name as an explicit classification field; current P3 samples and profile tests cover the known QDII/bond/index paths.
- External MiMo/GLM aggregate review panes were not usable in this controller run: MiMo was polluted by user `.tmux.conf` input, and GLM was occupied by an older long-running review context. This controller review therefore serves as the aggregate gate evidence for this run.
