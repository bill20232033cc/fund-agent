## Findings

未发现实质性问题。

逐项审查结论：

1. **Metadata 通过 documents.source_metadata_json 和 ParsedAnnualReport.metadata 持久化**: `cache.py:524-525` 计算有效元数据时使用 `source_metadata or report.metadata.source` 优先取显式参数，然后 `_normalize_report_source_metadata` 用 `replace()` 对齐 report 的 `metadata.source`，documents 行和 parsed payload 在同一 SQLite 事务中写入（`cache.py:532-586`），不会静默分叉。测试 `test_save_parsed_report_aligns_explicit_source_metadata_with_payload` 验证了显式 source_metadata B 覆盖 report 内嵌的 metadata A 后两处一致。

2. **Legacy parsed payload 和 legacy documents row 兼容**: `ParsedAnnualReport.from_dict()` 在 `models.py:568-569` 接受缺失的 metadata key，`AnnualReportMetadata.from_dict(None)` 返回空默认实例。`_ensure_documents_source_metadata_column` 通过 `PRAGMA table_info` + `ALTER TABLE ADD COLUMN` 增量加列。`_get_pdf_entry_sync` 对 `source_metadata_json` 为 null 的行返回 `source_metadata=None`。测试 `test_cache_loads_legacy_documents_row_without_source_metadata` 和 `test_legacy_parsed_report_without_metadata_loads_with_empty_metadata` 分别验证。

3. **无 adapter-wide 可变 metadata 状态**: `AnnualReportPdfAdapter` 没有 `_latest_source_metadata` 或类似实例字段。`fetch_pdf` 返回 frozen `AnnualReportPdfFetchResult(pdf_path, source_metadata)`。`fetch_pdf_path` 委托给 `fetch_pdf` 后只取 `pdf_path`。并发安全由 per-call immutable result 保证。

4. **AnnualReportPdfFetchResult per-call 安全性和并发测试有效性**: `_ConcurrentMetadataAwareLoader` 使用 `asyncio.Event` 屏障强制两个 `fetch_pdf` 调用交错执行。`test_repository_concurrent_loads_do_not_cross_attach_source_metadata` 通过 `asyncio.gather` 并发调用两个不同基金的 `load_annual_report`，断言每个返回的 `ParsedAnnualReport.metadata.source` 和 cache entry 的 `source_metadata` 均属于各自的基金。这直接回归测试了 P7S4-PR-1 所识别的并发 provenance corruption 风险。

5. **Metadata-aware vs legacy loader 行为**: `_get_metadata_aware_loader`（`repository.py:204-225`）依次检查 `fetch_pdf_path`、`parse_pdf`、`fetch_pdf` 是否为 coroutine function。Repository 主路径（`repository.py:346-361`）优先调用 `fetch_pdf`；若 loader 不支持则 fallback 到 `fetch_pdf_path` 并设 `source_metadata=None`。测试 `test_repository_metadata_aware_loader_preferred_over_legacy_fetch_pdf_path` 和 `test_repository_legacy_fetch_pdf_path_loader_gets_empty_source_metadata` 分别验证两条路径。

6. **force_refresh 覆盖**: `repository.py:323` 和 `repository.py:339` 两处 `if not force_refresh` 跳过 parsed cache 和 PDF entry 缓存。force_refresh 时直接 fetch、record、parse、save，metadata 全部使用新获取值。测试 `test_repository_force_refresh_overwrites_source_metadata` 验证 metadata_a 被 metadata_b 覆盖且 cache entry 也更新。

7. **Eastmoney fallback metadata 不伪造 EID ID**: `_eastmoney_fallback_metadata` 构造 `source="eastmoney"`, `fallback_used=True`，所有 EID 特有字段（`fund_id`, `upload_info_id`, `upload_info_detail_id` 等）保持 `None`。测试 `test_repository_attaches_eastmoney_fallback_metadata_on_fresh_fetch` 逐字段断言。

8. **Parsed cache hit / PDF cache hit provenance**: Parsed cache hit 时 `repository.py:326-334` 设 `parsed_cache_hit=True` 并保留原 `pdf_cache_hit`。PDF cache hit 时 `repository.py:373-379` 设 `pdf_cache_hit=True`, `parsed_cache_hit=False`。Fresh fetch 时两者均为 False。测试 `test_repository_parsed_cache_hit_retains_metadata` 和 `test_repository_pdf_cache_hit_uses_cached_source_metadata` 验证。

9. **无 Service/UI/Engine/CLI source 泄漏**: grep 确认 Service/UI/Engine 目录无 documents.sources、documents.cache、documents.repository、EidAnnualReportSource、AnnualReportSourceOrchestrator、AnnualReportDocumentCache、FundDocumentRepository 的 import。

10. **公共契约/export 一致性**: `AnnualReportSourceMetadata`、`AnnualReportSourceName`、`AnnualReportCacheProvenance`、`AnnualReportMetadata`、`AnnualReportPdfFetchResult` 全部定义在 `models.py`。`sources.py:19` 从 `models.py` import 这两个符号，不再有重复定义。无循环 import。所有新增类都是 `frozen=True, slots=True` dataclass。

11. **Plan 要求的 14 项测试全部存在**: 5 项 cache 测试（`test_cache_persists_pdf_source_metadata`, `test_cache_loads_legacy_documents_row_without_source_metadata`, `test_parsed_report_payload_round_trips_metadata`, `test_legacy_parsed_report_without_metadata_loads_with_empty_metadata`, `test_save_parsed_report_aligns_explicit_source_metadata_with_payload`）和 9 项 repository 测试（EID metadata, Eastmoney fallback, parsed cache hit, PDF cache hit, legacy cache, force refresh, metadata-aware preferred, legacy empty, concurrent no cross-attach）全部匹配 plan 要求。

## Residual Risk

- `save_parsed_report` 中 `source_metadata or report.metadata.source` 使用 Python `or` 语义：若调用方显式传入一个所有字段为 None 的 `AnnualReportSourceMetadata()` 实例，它不会 fallback 到 `report.metadata.source`。当前所有调用点都传入 None 或有实质内容的 metadata，风险极低。
- `PARSED_REPORT_SCHEMA_VERSION` 未递增。旧 payload 无 metadata 字段时正确加载，但如果未来 metadata schema 发生不兼容变更，缺少版本区分可能增加迁移复杂度。
- `_source_metadata_from_json` 在 JSON 格式非法时 raise ValueError，该异常会从 `get_pdf_entry` 传播至 repository。当前 metadata 写入由代码控制，格式可信，但如果人工编辑了 SQLite 行可能导致该条目不可读。
- 测试未覆盖 `_ensure_documents_source_metadata_column` 在已有列时的幂等行为（代码通过 `PRAGMA table_info` 检查已实现幂等，但无直接测试）。
