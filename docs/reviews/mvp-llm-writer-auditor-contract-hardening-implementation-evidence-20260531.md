## Changes

- Writer prompt/parser:
  - Added fixed section requirements for chapters 1-6: `### 结论要点`, `### 详细情况`, `### 证据与出处`.
  - Required every `required_output_items` entry to use exact marker `<!-- required_output:<item> -->`.
  - Added writer stop reasons: `missing_required_structure`, `missing_required_output_marker`, `unknown_anchor`, `response_too_long`, `response_incomplete`.
  - Added fail-closed handling for `finish_reason=length/max_tokens/content_filter`.
  - Kept unknown missing marker, invalid marker, evidence line without anchor, forbidden trading advice as contract violations.
- Auditor line protocol/calibration:
  - Programmatic audit now requires exact required output marker; bare item text no longer passes.
  - LLM auditor prompt now specifies exact `PASS|chapter|no issues` and `SEVERITY|LOCATION|MESSAGE` line protocol.
  - LLM parse failure remains blocked with stable `llm:parse_failure` and repair hint text.
  - Candidate/non-asserted facet disclaimer can pass; asserted candidate facet remains blocking with one issue per unique facet.
- Repair/regenerate:
  - Added typed `ChapterRepairContext` with explicit `attempt_index`, `previous_issue_ids`, `previous_messages`, `required_corrections`.
  - Regenerate attempts now rebuild writer input with previous audit issue context.
  - Required corrections are generated by deterministic rule/location/message mapping; unknown messages are sanitized and length-capped.
  - No `extra_payload` was introduced.
- Timeout/provider runtime classification:
  - Added `LLMProviderTimeoutError` and `LLMProviderNetworkError`.
  - Service maps provider runtime categories to `llm_timeout`, `llm_rate_limited`, `llm_malformed_response`, `llm_network_error`; unknown exceptions remain `llm_exception`.
- Docs decision:
  - Updated `fund_agent/fund/README.md` because the Fund writer/auditor public contract changed.
  - Did not update `tests/README.md`; test running conventions did not change.
