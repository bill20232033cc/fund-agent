- **实际行为**: 当前所有已注册 processor（`ActiveFundAnnualProcessor`、测试替身）的 `__init__` 均为 no-op，因此不触发此场景

- **建议**: 未来 slice 添加 processor 构造逻辑时，在 `resolve()` 中捕获异常并转为 typed error

#### 2-LOW — `_blocked_result` 的 contract_status 依赖 gap_code 命名约定

- **入口/函数**: `_blocked_result()` line 632
- **文件(行号)**: `active_annual.py:632`
- **输入场景**: 未来新增不以 `unsupported_` 开头的阻断 gap_code（如当前的 `input_type_mismatch`）
- **实际分支**: `"unsupported" if gap_code.startswith("unsupported_") else "blocked"`

- **建议**: 未来可用显式映射表替代 startswith 检查
