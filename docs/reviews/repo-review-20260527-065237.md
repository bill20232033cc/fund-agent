### F3-未修复-[低]-来源元数据反序列化会把字符串 `"false"` 当成 `fallback_used=True`

- **入口/函数**: `AnnualReportDocumentCache._source_metadata_from_json()` / `ParsedAnnualReport.from_dict()` 反序列化来源元数据后，`FundDataExtractor.extract()` 投影为 `source_provenance`
- **文件(行号)**: `fund_agent/fund/documents/models.py:96-128`、`fund_agent/fund/documents/cache.py:114-138`、`fund_agent/fund/source_provenance.py:126-176`
- **输入场景**: cache、迁移脚本、人工修复或外部工具写入 JSON：`{"source":"eastmoney","fallback_used":"false"}`；该字符串在 JSON 层不是布尔值，但很容易出现在手工维护或跨工具导出的元数据中
- **实际分支**: `AnnualReportSourceMetadata.from_dict()` 使用 `bool(payload.get("fallback_used", False))`，Python 中 `bool("false")` 为 `True`；随后 `project_public_source_provenance()` 进入 fallback-used 分支
- **预期行为**: `fallback_used` 是 provenance 安全字段，应只接受布尔 `true/false`，或显式解析有限字符串；非法类型应降级为空元数据或 fail-closed，不应被 Python truthiness 静默改义
- **实际行为**: 字符串 `"false"` 被静默转换为 `True`，公共 provenance 输出 `fallback_used=True`、`fallback_eligibility=unknown_public_metadata_absent`
- **直接证据**: `models.py:127` 的 `bool(...)` coercion；`cache.py:130-137` 会把 SQLite 中的 JSON 字段交给 `AnnualReportSourceMetadata.from_dict()`；可复现命令输出为 `True True unknown_public_metadata_absent fallback_used_primary_failure_category_absent`
- **影响**: 当前生产写入路径用 `to_dict()` 写入真实 bool，因此不是常规 happy path bug；但一旦旧缓存/手工缓存/迁移数据出现字符串布尔值，就会污染 source provenance evidence rerun，把“没有 fallback”误读为“fallback used 但缺 primary_failure_category”，从而误导小样本来源覆盖判断
- **建议改法和验证点**: 增加 `_optional_bool()` 或 strict bool parser：仅接受 `bool`，或显式映射 `"true"/"false"`；其他类型抛 `ValueError` 让 cache 按现有逻辑降级为空元数据。补测试覆盖 `"false"`、`"true"`、`0/1` 或非法字符串的预期。验证点：上述复现命令不再输出第一个 `True`
- **修复风险（低/中/高）**: 中，需确认是否已有历史缓存依赖字符串布尔；建议作为 source provenance 后续 hardening 小修，而不是顺手混入报告模板 gate
- **严重程度（低/中/高/严重）**: 低
