## Findings

### PR-001-未修复-中-报告差异测试要求过窄，会迫使实现写脆弱或不真实的 fixture
- **位置**: `S2. Wire Renderer` / Tests
- **问题类型**: 测试缺口 / 不可直接实施
- **当前写法**: plan 要求 “Active and index reports differ only in intended lens-dependent slots and fund type text for the fixture.”
- **反例/失败场景**: 为了让 active/index fixture 自洽，测试需要同时改变 `basic_identity.value["classified_fund_type"]` 和 `stress_test_result.fund_type`。但当前 renderer 第 6 章会输出 `input_data.stress_test_result.fund_type`，因此两份报告除 Chapter 0/1 lens slot 和基金类型文本外，还会在 Chapter 6 压力测试行不同。如果测试只改 `basic_identity` 不改 `stress_test_result`，报告差异会更少，但输入事实变成“指数基金身份 + active_fund 压力测试”，不是一个真实可接受 fixture。
- **为什么有问题**: P8-S2 的目标是验证 Chapter 0/1 lens application 和结构/audit 不回归，不是证明整篇 Markdown 只有两个字符串不同。当前测试要求把 implementation agent 推向两种坏选择：写不真实输入，或写一个会因现有非目标字段变化而失败的 brittle whole-report diff。
- **直接证据**:
  - `fund_agent/fund/template/renderer.py` Chapter 6 当前渲染：`压力测试：基金类型 {input_data.stress_test_result.fund_type}`。
  - `tests/fund/template/test_renderer.py` fixture 当前 `_stress_test()` 默认 `fund_type="active_fund"`。
  - plan 同时要求 active/index 输出差异测试和 preserving existing report/audit behavior。
- **影响**: 实施 Agent 跑偏 / 生成脆弱测试 / review 不可验收 / 后续为非目标差异返工。
- **建议改法和验证点**:
  - 把该测试要求改成“assert exact Chapter 0/1 slot text differs as intended, and assert unchanged structural/audit invariants”；不要要求 whole-report 只存在限定差异。
  - 如需比较差异，限定在 parsed `chapter_blocks[0]` 和 `chapter_blocks[1]` 的相关 required item 行。
  - 对输入自洽性要求写清：当 fixture fund type 改为 `index_fund` 时，相关结构化输入如 `stress_test_result.fund_type` 可以同步改，但不把 Chapter 6 文本差异纳入本 slice 验收。
- **修复风险（低）**: 只需收窄测试验收口径，不改变实现方向。
