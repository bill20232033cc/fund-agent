### 审查证据摘要

逐项验证了 controller judgment 要求的审查重点：

**1. `candidate_boundary=None` 不能成为 source truth proof**

- `_validate_source_truth_admission` (`fund_disclosure_processor.py:686-732`) 要求 content intermediate 必须持有 `FundDisclosureSourceTruthAdmissionProof` 实例且通过完整身份校验，才返回 `None`（proof 通过）。
- `candidate_boundary=None` 本身只是 admission helper 的必要条件之一，不进入 source-truth proof 判定。
- 测试 `test_source_truth_admission_requires_positive_proof`（行 343）用 `candidate_boundary=None, source_truth_admission=None` fixture 验证了 `source_truth_admission_missing` gap 被追加到所有字段族。

**2. proof missing/invalid 必须 fail closed 且无 public value/anchors**

- `_with_source_truth_admission_gap` (`fund_disclosure_processor.py:2773-2814`) 将 status 强制设为 `"missing"`、value 设为 `{}`、anchors 设为 `()`，并追加 source_truth_admission gap。
- 测试 `test_source_truth_admission_requires_positive_proof` 验证了 `product.value == {}`、`product.anchors == ()`、`source_truth_admission_missing in _gap_codes(product)`。
- 测试 `test_source_truth_admission_rejects_identity_mismatch`（行 378）用 `fund_code="110011"` 不匹配 fixture 验证了 `source_truth_admission_invalid` gap 被追加。

**3. candidate evidence diagnostic 是否被错误抹掉或提升**

- `_with_source_truth_admission_gap` 保留了 `family.candidate_evidence`（行 2813），candidate evidence 记录未被抹掉。
- candidate evidence 的 `source_boundary` 仍为 `"candidate_only"`、`candidate_only=True`、`field_correctness_status="not_proven"`、`source_truth_status="not_proven"`，未被提升。
- 测试 `test_source_truth_admission_requires_positive_proof` 的 `assert product.candidate_evidence`（行 375）确认了 candidate evidence 仍存在。

**4. candidate-boundary blocked、default parsed route、non-active explicit FDD 是否回归**

- candidate-boundary blocked：`admit_disclosure_intermediate` (`fund_disclosure_dispatch.py:119-125`) 对 `candidate_boundary is not None` 返回 `admitted=True, contract_status="blocked"`，processor 提取中 `source_truth_gap_code` 赋值条件 `admission.contract_status != "blocked"` 阻止了 source-truth validation 运行。现有 candidate-boundary 测试全部通过。
- default parsed route：`test_default_active_fund_still_uses_parsed_annual_report_processor_path`、`test_default_extract_does_not_resolve_fund_disclosure_processor` 通过。
- non-active explicit FDD：`test_explicit_disclosure_non_active_fund_raises_no_legacy_fallback`、`test_default_non_active_without_disclosure_preserves_legacy_path` 通过。
- 全部 133 个测试通过（`tests/fund/processors/test_fund_disclosure_processor.py` + `tests/fund/test_data_extractor.py`）。

**5. 是否越权实现 Slice B 或扩 EvidenceSourceKind**

- 未实现 `product_essence.v1` public extraction。proof-positive fixture 仍返回 `missing`（测试 `test_source_truth_admission_accepts_repository_loaded_identity_proof`，行 411）。
- 新增的 `source_truth_unverified` 是 `FundExtractionSourceBoundary`（内部 gap 边界），不是 `EvidenceSourceKind`。`EvidenceSourceKind` 未被修改。
- proof 的 `source_kind` 固定为 `"annual_report"`，未引入 FDD 或 candidate source kind。

### 代码路径走读

**`extract()` 主链路**（`fund_disclosure_processor.py:448-556`）：

1. `supports()` 校验 active_fund + annual_report + fund_disclosure_document.v1 — 不变
2. `isinstance(intermediate, FundDisclosureDocumentIntermediate)` 协议检查 — 不变
3. `_check_identity()` 身份一致性校验 — 不变
4. `admit_disclosure_intermediate()` admission 判定 — 不变
5. **新增**：`source_truth_gap_code = _validate_source_truth_admission(intermediate, context)`（仅当 `admission.contract_status != "blocked"` 时执行）
6. `_field_families_for_intermediate(intermediate, source_provenance, source_truth_gap_code=source_truth_gap_code)` — 新增 keyword 参数
7. 返回 result — 不变

**`_validate_source_truth_admission()`**（行 686-732）：

1. `_content_intermediate_or_none()` 判断是否为 content intermediate — 非 content 返回 `None`（跳过 validation）
2. `proof is None` → `source_truth_admission_missing`
3. `not isinstance(proof, FundDisclosureSourceTruthAdmissionProof)` → `source_truth_admission_invalid`
4. `candidate_boundary is not None` / `failure_class is not None` / `source_provenance is None` → `source_truth_admission_invalid`
5. 身份一致性比对（proof ↔ context ↔ intermediate）→ 不匹配返回 `source_truth_admission_invalid`
6. 全部通过 → `None`

**`_with_source_truth_admission_gap()`**（行 2773-2814）：

- 保留原 field family 的 `field_family_id`、`chapter_ids`、`source_provenance`、`candidate_evidence`
- 强制 `status="missing"`、`value={}`、`anchors=()`
- 追加 source_truth_admission gap 到 `gaps` tuple

## Residual Risk

- Slice A 不实现 `product_essence.v1` public extraction（Slice B 范围），proof-positive fixture 当前返回 `missing` 是预期行为。
- 当前无生产 FDD producer 能创建 `FundDisclosureSourceTruthAdmissionProof`；生产调用方无 proof 时将 fail closed。
- `_content_intermediate_or_none()` 使用 `hasattr` 结构检查而非 `isinstance` runtime_checkable protocol 检查，行为等价但 cast 后的类型安全依赖运行时属性存在性。
