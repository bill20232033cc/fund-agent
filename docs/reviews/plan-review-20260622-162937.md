## Findings

### 1-未修复-高-plan 需要不存在的 table-section 事实，可能为了 V2 pass 伪造 section proof

- **位置**: `Implementation Decision` step 4 and `Success Signal`
- **问题类型**: 契约缺失 / 测试缺口 / 架构边界
- **当前写法**: plan 要从 `ParsedAnnualReport.tables` 选择 table row，并构造包含 `section_id`、`page_number`、`table_id`、`row_locator` 的 anchor，目标是让 V2 `evidence_confirm_overall_status="pass"`。
- **反例/失败场景**: 当前 `ParsedTable` 只有 `page_number`、`table_index`、`headers`、`rows`，没有 `section_id` 或 table-to-section mapping。implementation agent 为了满足 V2 pass，可能从 `parsed_report.sections` 任取一个 section_id 填入 anchor。materializer 会把该 section_id 原样写入 reference，而不是证明 table 属于该 section。V2 proof_boundary 只比较 anchor/reference 内同一个 section_id，于是会错误 pass。
- **为什么有问题**: 这会把“表格 locator 可 materialize”伪装成“表格有可信 section locator”，违反 EC-P2 不证明字段正确性/语义 entailment/source truth 的边界，也会让 V2 pass 依赖未由 `ParsedAnnualReport` 承载的事实。
- **直接证据**:
  - `fund_agent/fund/documents/models.py` `ParsedTable` fields are `page_number`, `table_index`, `headers`, `rows`; no section field.
  - `fund_agent/fund/evidence_confirm_sources.py::_anchor_preflight_issue()` requires `anchor.section_id` to exist in `parsed_report.sections` before table excerpt is built.
  - `fund_agent/fund/evidence_confirm_sources.py::_build_reference_for_anchor()` emits `section_id=anchor.section_id`; table excerpt path does not validate table-to-section relationship.
  - `fund_agent/fund/evidence_confirm.py::_precision_issue()` still requires `reference.section_id` plus page/table/row precision for annual-report references.
- **影响**: Implementation may create a V2 pass that is stronger than the current data model can justify; later Service/quality-gate integration could treat this as source/PDF Evidence Confirm proof when it only proved table-row formatting.
- **建议改法和验证点**:
  - Do not require V2 `pass` for EC-P2 pathway success unless table-section mapping is added by a separate reviewed parser/model gate.
  - Add a separate runner/script `pathway_status` or `source_pdf_pathway_status` that can pass when repository load, source metadata admission, reference materialization, and non-blocking V2 result are present.
  - Keep strict `evidence_confirm_overall_status="warn"` visible and preserve `status="fail"` or rename it to `strict_status`.
  - Add tests proving section-only smoke yields `pathway_status="pass"` but `evidence_confirm_overall_status="warn"` and `field_correctness_proven=false`.
- **修复风险（低/中/高）**: 中
- **严重程度（低/中/高/严重）**: 高
