### 001-未修复-中-source_field_path format is underspecified
- **位置**: plan §4 lines 64-67, §7 lines 124-132
- **问题类型**: 不可直接实施 / 测试缺口 / 契约缺失
- **当前写法**: plan requires deduplication by exact `source_field_path`, but does not define path formats for section, paragraph, table, or cell records.
- **反例/失败场景**: Implementation uses `section:0`, another future selector uses `sections[0]`, and tests only assert non-empty paths. Deduplication and downstream evidence comparison become unstable even though each implementation is internally deterministic.
- **为什么有问题**: `source_field_path` is the stable locator identity for candidate evidence. Without a reviewed format, implementation workers invent contract details that later gates may depend on.
- **直接证据**: plan lines 64-67 require exact-path dedupe; `FundCandidateEvidenceRecord` requires non-empty `source_field_path`; no path grammar is specified.
- **影响**: Unstable evidence records, weak tests, avoidable future migration, selector outputs hard to compare across fields.
- **建议改法和验证点**: Add binding path formats before implementation: `sections[{index}]`, `paragraph_blocks[{index}]`, `table_blocks[{index}]`, and `table_blocks[{table_index}].cells[{cell_index}]`. Tests should assert exact path strings for at least one section/table/cell case.
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 中
