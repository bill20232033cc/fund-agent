# Aggregate Deepreview: Extractor Output Repository

## Findings

### 001-未修复-中-save 返回 payload 与落盘/load payload 的 sequence 形态不一致

- **Entry point**: `ExtractorOutputRepository.save(...)` and `ExtractorOutputRepository.load(...)`
- **Triggering input**: 保存任意含 tuple/list 字段的 `StructuredFundDataBundle`，例如 `ExtractedField.anchors` 或 `classification_basis`
- **Actual branch**: `_jsonable(...)` 对 `Sequence` 返回 `tuple(...)`；`json.dumps(...)` 会把 tuple 写成 JSON array；`load()` 再从 JSON 读回 Python `list`
- **Expected behavior**: Repository record 的 `bundle_payload` 应是稳定 JSON-shaped payload；同一文件刚 save 返回的 `record.bundle_payload` 与 load 返回的 `record.bundle_payload` 不应因为入口不同而出现 tuple/list 形态差异
- **Actual behavior**: `save()` 返回的 record 里 sequence 是 tuple；`load()` 返回的 record 里同一字段是 list
- **Direct evidence**: `fund_agent/fund/extractor_output_repository.py` 中 `_jsonable(...)` 的 `Sequence` 分支返回 `tuple(...)`；`load()` 通过 `json.loads(...)` 返回 list
- **Impact**: 后续 Service 或 consumer 如果使用 save 返回值做即时索引/传递，使用 load 返回值做恢复，会面对不同 Python 形态；这会给多年分析或 LLM route 的输入层制造非必要的 version skew 和测试分支
- **Fix direction**: 把 `_jsonable(...)` 的 `Sequence` 分支改为返回 `list[...]`，并新增测试断言 `save()` 返回 payload 与 `load()` 返回 payload 在 sequence 形态上保持一致
- **Fix risk**: 低
- **Severity**: 中
