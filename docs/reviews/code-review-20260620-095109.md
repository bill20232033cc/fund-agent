### 001-未修复-中-strategy 段落匹配包含正文文本导致潜在误命中

- **入口/函数**: `_manager_profile_paragraph_matches_strategy()` (fund_disclosure_processor.py:1473-1497)
- **文件(行号)**: fund_agent/fund/processors/fund_disclosure_processor.py:1490-1497
- **输入场景**: 管理层讨论等非策略 heading 下的段落，其正文文本恰好包含 "投资策略"、"运作分析" 等 heading token
- **实际分支**: `_matches_any_token` 同时检查 `paragraph.heading_path`、`paragraph.text_normalized`、`paragraph.text_raw`；正文命中即返回 True
- **预期行为**: 计划 Section 5.2 要求 "stable paragraph blocks whose heading_path, text_normalized, or text_raw is under" 允许 heading；但实现将 heading 判断和正文内容判断合并为同一个 OR，导致非策略 heading 下的段落若正文包含策略 heading token 会被误匹配
- **实际行为**: 正文包含 "投资策略" 的段落（如 "我们改变了传统的投资策略方向"）在非策略 heading 下仍会被抽取为 `strategy_summary`
- **直接证据**: fund_disclosure_processor.py:1490-1497 — `text_normalized` 和 `text_raw` 直接参与 heading token 匹配
- **影响**: 局部行为错误 — 非策略段落内容可能污染 `manager_strategy_text`；实际披露文档中此场景概率较低，但不符合计划的严格 heading-gated 语义
- **建议改法和验证点**: 将 `_manager_profile_paragraph_matches_strategy` 拆分为两步：先检查 `heading_path` 是否命中允许 heading，若未命中再检查 `text_normalized`/`text_raw` 是否本身就是 heading（即段落标题行）；或仅对 `heading_path` 做 heading 匹配，正文文本不做 heading token 匹配。验证点：构造非策略 heading 下正文包含策略 token 的测试用例，确认不被误抽取
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 中

### 002-未修复-低-test 断言对缺失 key 的检查是空真断言

- **入口/函数**: `test_manager_profile_source_truth_partial_when_required_groups_missing()` (test_fund_disclosure_processor.py:2829)
- **文件(行号)**: tests/fund/processors/test_fund_disclosure_processor.py:2877
- **输入场景**: 只有 `manager_strategy_text` 被抽取的 partial 场景
- **实际分支**: `family.value` 只含 `schema_version` 和 `manager_strategy_text`，`family.value.get("portfolio_managers")` 返回 `None`（dict 默认值）
- **预期行为**: 断言应验证缺失 top-level key 不出现在 value 中（即 key 不存在），而非检查 `get()` 返回 None
- **实际行为**: `assert all(family.value.get(key) is None for key in ("portfolio_managers", "turnover_rate"))` 对不存在的 key 返回 None 是空真；即使实现错误地将缺失 key 设为 `None`（如 `{"portfolio_managers": None}`），此断言仍通过
- **直接证据**: test_fund_disclosure_processor.py:2877 — `dict.get()` 对不存在的 key 返回 None
- **影响**: 测试覆盖力减弱 — 无法区分 "key 不存在" 和 "key 存在但值为 None"；计划要求缺失 top-level 用 gaps 表示，不应出现在 value 中
- **建议改法和验证点**: 改为 `assert "portfolio_managers" not in family.value` 和 `assert "turnover_rate" not in family.value`，或直接用已有的 `set(family.value) == {"schema_version", "manager_strategy_text"}` 断言（line 2871 已覆盖）
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 低
