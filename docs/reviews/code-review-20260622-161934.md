### 1-未修复-高-live sample 绕过 runner 的 repository failure classification 和 safe JSON 输出

- **入口/函数**: `run_authorized_sample()`
- **文件(行号)**: `scripts/evidence_confirm_ec_p2_live_sample.py:149`
- **输入场景**: 执行已授权 live 命令，且真实 `FundDocumentRepository.load_annual_report("004393", 2025, force_refresh=True)` 因网络、PDF、EID schema、identity 或 integrity 问题失败。
- **实际分支**: 脚本先在 `run_authorized_sample()` 内直接创建 `FundDocumentRepository()`，然后在 runner 外部调用真实 `repository.load_annual_report(...)`。`_main_async()` 只捕获 `LiveProjectionUnavailableError`，不捕获 repository 边界异常。
- **预期行为**: EC-P2 live sample 应通过 repository-bounded runner 的失败分类路径 fail-closed，并输出安全标量 JSON；不能把 repository/source/PDF 异常以 traceback 形式泄漏到命令输出，也不能绕过 `not_found / unavailable / schema_drift / identity_mismatch / integrity_error / ambiguous_repository_failure` 分类。
- **实际行为**: 真实 repository 失败发生在 `run_repository_bounded_evidence_confirm()` 之前，runner 的 `except Exception` 分类分支不会执行；异常会从 `_main_async()` 逃逸，CLI 输出 Python traceback，而不是 safe scalar JSON。
- **直接证据**:
  - `scripts/evidence_confirm_ec_p2_live_sample.py:162-169` 直接实例化真实 `FundDocumentRepository` 并调用 `load_annual_report`。
  - `scripts/evidence_confirm_ec_p2_live_sample.py:171-179` 随后把已加载的 parsed report 包装成 `_LoadedReportRepository` 再传给 runner，runner 只调用 wrapper，不调用真实 repository。
  - `scripts/evidence_confirm_ec_p2_live_sample.py:327-330` 只捕获 `LiveProjectionUnavailableError`，不捕获 repository load failure。
  - `fund_agent/fund/evidence_confirm_sources.py:287-300` 的 repository failure classification 只覆盖 runner 内部调用 `load_annual_report` 抛出的异常；当前 live script 的真实失败不会走到这里。
- **影响**: 已授权 live/PDF sample 的 failure path 可能输出 traceback，破坏 EC-P2 “safe scalar JSON only”和 fail-closed classification 语义；同时 live command 不能证明真实 repository load 经由 runner 边界受控。
- **建议改法和验证点**:
  - 改为让 runner 执行真实 repository load，并提供受控的 post-load projection hook/factory；或新增 runner 结果携带 parsed-report-derived smoke projection所需的最小安全通道，避免脚本在 runner 外部先 load。
  - 如果保持脚本先 load，则必须至少把该 load failure 映射为与 runner 一致的安全 JSON 分类，但这仍不能证明 runner 真实 repository load 边界。
  - 增加 no-live 单测：fake real repository 在脚本 live path 的首次 load 抛出 `AnnualReportSourceUnavailableError` 类名异常时，CLI/helper 返回 safe JSON，不 traceback，并保留 failure category。
  - 增加正向单测：真实 repository-like fake 是传入 runner 的 repository，runner call count 表示真实 load 发生在 runner 内。
