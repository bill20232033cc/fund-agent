## Findings

### 1-未修复-中-CLI auto-valuation sample lacks an injectable deterministic thermometer path

- **位置**: Slice S6-1, item 2: "add at least one auto-valuation CLI sample for exact `000300` or `000905` using deterministic fake services/cache"
- **问题类型**: 不可直接实施 / 测试缺口 / 契约缺失
- **当前写法**: Plan requires a CLI e2e auto-valuation sample but only says to use deterministic fake services/cache. It does not specify whether the implementation should inject the fake thermometer into `FundAnalysisService`, monkeypatch `ThermometerService`, or avoid the existing P3 CLI e2e harness.
- **反例/失败场景**: Implementation agent extends `tests/fund/integration/test_p3_cli_e2e_matrix.py` by simply removing `--valuation-state` from a sample case. That test's current harness monkeypatches `cli.FundAnalysisService` to return a preconstructed `FundAnalysisService(extractor=...)` without injecting a thermometer fake. The service then uses the default `IndexThermometerService`, which may read cache or call akshare depending on local state. The intended deterministic CLI e2e becomes environment-dependent or accidentally live.
- **为什么有问题**: P19-S6's core boundary is deterministic production validation with live smoke kept opt-in. Leaving the fake thermometer injection point unspecified risks violating that boundary exactly where the plan tries to prove auto valuation. It also forces the implementation agent to redesign test architecture.
- **直接证据**:
  - `fund_agent/services/fund_analysis_service.py` supports injection through `FundAnalysisService(..., thermometer_service=...)`.
  - `_resolve_valuation_state()` calls `self._thermometer_service.run(ThermometerRequest(... index_code=target.index_code))` when request `valuation_state` is `None`.
  - `tests/fund/integration/test_p3_cli_e2e_matrix.py` currently constructs `FundAnalysisService(extractor=...)` and all sample `cli_args` include explicit `--valuation-state fair`, so the automatic path is not exercised there.
- **影响**: Implementation may introduce live-network dependence in deterministic tests, fail intermittently, or fail to exercise the intended CLI auto-valuation path at all.
- **建议改法和验证点**:
  - Update the plan to require an explicit fake thermometer service class in the CLI e2e test harness.
  - The fake should record `ThermometerRequest.index_code` and return a `ThermometerReading` for the selected exact benchmark.
  - The auto-valuation CLI sample must omit `--valuation-state`, use the injected fake thermometer, and assert the fake was called with `000300` or `000905`.
  - Add a negative CLI or Service test proving unsupported / ambiguous cases do not call the fake thermometer.
- **修复风险（低）**: This is a plan clarification and uses an existing injection seam.
- **严重程度（中）**: Without it, the core P19-S6 deterministic validation goal can be missed or turned into accidental live smoke.
