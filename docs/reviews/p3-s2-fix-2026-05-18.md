## Accepted Finding

GLM identified a low-severity theoretical false-positive risk in all-market parsing: when text near `全市场温度` contains a non-temperature number before the actual degree-marked value, `_decimal_after_labels(...)` could select the nearby number.

MiMo also noted that `_degree_after_heading(...)` only accepted `°`, not `℃`.

## Fix

- `_parse_market_temperature(...)` now prefers `_degree_after_heading(...)` before `_decimal_after_labels(...)`.
- `_degree_after_heading(...)` now accepts both `°` and `℃`.
- Added `test_parse_thermometer_pages_prefers_degree_market_value_over_nearby_number`.
