## Findings

### Blocking: Ch3 typed must_not_cover can be bypassed by putting a positive claim on the same line as an anchor marker

- Severity: blocking correctness / contract enforcement.
- Entry point: `audit_chapter_programmatic()` on a typed Ch3 draft with missing or unreviewed behavior/style evidence.
- Triggering input: a draft line shaped like `<!-- anchor:<allowed_anchor_id> --> 言行一致性判断：言行一致。`.
- Expected behavior: under Slice 0 calibration and Slice 4 accepted semantics, `言行一致` and quasi-positive equivalents must block when Ch3 actual behavior/style evidence is missing, unavailable or unreviewed, unless the phrase is only in a narrow allowed context. Slice 0 explicitly says required labels followed by a positive conclusion in the same line are not allowed, and anchor captions may not be body conclusions.
- Actual behavior: `_audit_ch3_style_must_not_cover_clause()` skips the entire line before sentence scanning whenever `_line_is_contract_or_anchor_metadata(line)` is true. `_line_is_contract_or_anchor_metadata()` treats any line whose stripped text starts with `<!-- anchor:` as metadata. The writer parser accepts the same marker anywhere a regex match occurs and does not require the marker to occupy the whole line, so a provider can place a normal positive conclusion after the marker and the Ch3 typed C2 check never inspects it.
- Direct evidence:
  - [chapter_auditor.py](/Users/maomao/fund-agent/fund_agent/fund/chapter_auditor.py:837) skips lines before checking Ch3 style phrases.
  - [chapter_auditor.py](/Users/maomao/fund-agent/fund_agent/fund/chapter_auditor.py:1175) classifies any line starting with anchor marker prefix as metadata.
  - [chapter_writer.py](/Users/maomao/fund-agent/fund_agent/fund/chapter_writer.py:1763) parses anchor markers with regex search over the whole text, so a marker followed by body text is accepted as an allowed anchor reference.
  - [slice0 evidence](/Users/maomao/fund-agent/docs/reviews/mvp-typed-template-contract-slice0-calibration-precondition-evidence-20260603.md:95) allows anchor caption only as metadata/caption, not body conclusion.
  - Existing Ch3 tests cover positive claims on normal body lines, but not marker/caption-line laundering.
- Impact: this weakens the main Slice 4 contract. The provider can satisfy anchor-marker parsing and still emit a forbidden positive reader takeaway in Ch3 under missing/unreviewed evidence, causing the programmatic audit to pass when it should fail closed.
- Fix direction: only skip exact metadata marker lines, not prefix-matching lines with trailing body text; add a regression test where a positive Ch3 consistency claim follows an allowed anchor marker on the same line and assert the typed C2 issue is emitted. If evidence-caption lines remain exempt, make that exemption deterministic and narrow enough that body conclusions after the marker/caption are still scanned.
- Fix risk: low. Tightening metadata skipping affects only lines that currently mix metadata and body text; proper standalone marker lines remain exempt.
