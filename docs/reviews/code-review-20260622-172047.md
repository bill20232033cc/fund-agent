## Findings

### 001-未修复-[中]-not_applicable semantic judgment is incorrectly escalated to warn under anchor_precision warning

- **入口/函数**: `confirm_semantic_entailment()` -> `_confirm_single_claim()` -> `_severity_for_judgment()`
- **文件(行号)**: `fund_agent/fund/evidence_confirm_semantic.py:451`
- **输入场景**: Deterministic V2 fact has only an `anchor_precision` warning, so `fact_result.hard_gate.status == "warn"`, and the injected semantic client returns `EvidenceEntailmentJudgment(status="not_applicable", reason_code="not_applicable")`.
- **实际分支**: `_severity_for_judgment()` skips only `contradicted` and `insufficient`, then sees `fact_result.hard_gate.status == "warn"` and returns `severity="warn"`.
- **预期行为**: Accepted plan maps valid client statuses explicitly: `not_applicable -> severity="info"`. The anchor_precision warning rule is intended for entailed semantic support, not for a semantic claim the client says is not applicable.
- **实际行为**: A semantic not-applicable judgment becomes a warning whenever deterministic V2 has a warning, so aggregate status becomes `warn` instead of `not_applicable`.
- **直接证据**: `_severity_for_judgment()` branches at `fund_agent/fund/evidence_confirm_semantic.py:451-457`; accepted plan lines 183-187 state `not_applicable -> severity="info"`.
- **影响**: 仅局部行为错误 / 错误状态。A caller can receive a semantic warning for a claim that the semantic client explicitly classified as not applicable, which overstates semantic review risk and breaks the accepted contract.
- **建议改法和验证点**: Add an explicit `if judgment.status == "not_applicable": return "info"` before applying the deterministic hard-gate warn rule. Add a regression test where anchor_precision warn plus semantic not_applicable aggregates to `not_applicable` and keeps client called once.
- **修复风险（低/中/高）**: 低
- **严重程度（低/中/高/严重）**: 中
