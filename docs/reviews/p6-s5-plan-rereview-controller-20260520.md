## Finding Reconciliation

### P6S5-PLAN-001 - accepted and fixed

Original finding:

- The plan classified multi-value `classified_fund_type` conflicts as `not_applicable`.
- That would allow a deterministic fund type contradiction to avoid FQ5/block, especially when the underlying field records are present and FQ2/FQ4 do not fire.

Fix applied to the plan:

- `classified_fund_type` multi-value conflict is now explicitly `mismatch`, not `not_applicable`.
- `_derive_contract_applicability(...)` must detect `fund_type_reason` containing `classified_fund_type 存在冲突值`, preserve the conflicting values, and skip manifest evaluation.
- Tests now separately require:
  - missing `classified_fund_type` -> `not_applicable`;
  - multi-value `classified_fund_type` -> `mismatch`;
  - score-derived mismatch -> FQ5/block with conflict reason preserved.

Controller judgment:

- The blocker is resolved.

## Accepted Implementation Boundary

P6-S5 implementation may proceed with these constraints:

- `extraction_score.py` may import public `CHAPTER_CONTRACT` and `ITEM_RULE` manifest helpers and write deterministic applicability facts into `score.json`.

- `ITEM_RULE` facts must be described as evaluator decisions only, not renderer compliance.
- No report Markdown parsing, LLM audit, Evidence Confirm, semantic NLP, or document repository access belongs in this slice.
