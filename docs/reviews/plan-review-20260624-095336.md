## Findings

### PR-001-未修复-高-S2 narrowing lacks a precise token-source and ambiguity contract

- **位置**: `Slice A3-S2: Coarse-reference Materializer Narrowing`, lines 212-222.
- **问题类型**: 契约缺失 / 不可直接实施 / 测试缺口
- **当前写法**: Plan says the narrowing helper may use "optionally fact value context if safely available", may construct an anchor-to-fact map from `ChapterFactProjection`, and should select a row when it contains the "deterministic V2 value token source used by the fact".
- **反例/失败场景**: A single anchor can be referenced by multiple facts or by a dict-valued fact with several material tokens. If implementation narrows a table to a row containing only one token, V2 may still fail for unmatched sibling tokens. If it searches for raw `str(fact.value)` or ad hoc values instead of V2 `_material_tokens`, it can create rows that appear plausible but do not match V2 semantics. If multiple facts share an anchor, the implementation may choose a row that satisfies one fact and breaks another.
- **为什么有问题**: The plan intends to preserve V2 as source truth, but does not specify whether S2 may import/reuse V2 token helpers, duplicate them, or introduce a new local token extraction helper. A2-S2 explicitly accepted `deterministic_v2_same_source_primitives`; implementing narrowing without that exact primitive risks creating a parallel matcher.
- **直接证据**:
  - Plan lines 212-216 leave fact value context optional and underspecified.
  - Plan line 222 requires V2 token source but does not name the function/API to use.
  - `fund_agent/fund/evidence_confirm.py` defines private `_material_tokens`, `_normalize_text` and `_token_matches_excerpt`.
  - `fund_agent/fund/evidence_confirm_value_diagnostics.py` imports those same private V2 primitives and declares `MATCH_SOURCE = "deterministic_v2_same_source_primitives"`.
- **影响**: Implementation Agent must redesign matching semantics during implementation, likely causing a parallel matcher, false row precision, or non-deterministic live re-evidence failures.
- **建议改法和验证点**:
  - Amend S2 to require reuse of V2 deterministic token primitives through an explicit private import from `evidence_confirm.py`, matching the accepted diagnostic helper pattern.
  - Define the narrowing contract: build `anchor_id -> facts` from `ChapterFactProjection`; narrow only when exactly one available non-derived fact references the anchor and all V2 material tokens for that fact are found in exactly one table row after V2 normalization/matching.
  - If zero, multiple, or partial-token rows match, preserve the existing table/section downgrade and informational issue.
  - Add tests for shared-anchor ambiguity and partial token match.
- **修复风险（低/中/高）**: 中
- **严重程度（低/中/高/严重）**: 高
