### 001-未修复-[中]-reference proof 未绑定章节 anchor/source identity，错误来源可关闭 E2/E3

- **入口/函数**: `confirm_chapter_evidence()` / `confirm_projection_evidence()` -> `_proof_references_for_fact()` -> `_reference_is_proof()` -> `_matched_anchor_ids()`
- **文件(行号)**: `fund_agent/fund/evidence_confirm.py:168`, `fund_agent/fund/evidence_confirm.py:206`, `fund_agent/fund/evidence_confirm.py:340`, `fund_agent/fund/evidence_confirm.py:360`, `fund_agent/fund/evidence_confirm.py:385`, `fund_agent/fund/evidence_confirm.py:417`
- **输入场景**: 一个非 derived 的 annual-report fact，例如 `structured.turnover_rate`，自身 `evidence_anchor_ids=("chapter-anchor:110011:2024:...",)`；调用方传入同一 `anchor_id` 的 `EvidenceConfirmReference(reference_kind="derived_calculation", source_kind="derived", document_year=2023, excerpt_text="换手率 120%")`，或传入 document_year/source metadata 与章节 anchor 不一致但 value token 命中的 reference。
- **实际分支**: `_proof_references_for_fact()` 只按 `fact.evidence_anchor_ids` 找到同名 reference；`_reference_is_proof()` 只校验 reference 自身满足 `candidate_only=False`、`source_truth_status=="proven"`、闭集 `reference_kind/source_kind` 配对；`_precision_issue()` 对 `derived_calculation` 直接返回 `None`；`_matched_anchor_ids()` 看到 token 出现在 excerpt 后返回 matched anchor。
- **预期行为**: Evidence Confirm phase 1 即使是 no-live helper，也应证明 reference 与 fact 自己的章节 anchor/source identity 相容。annual-report fact 不应被 `derived_calculation/derived` reference 关闭；annual-report reference 的 `document_year`、`source_kind` 和 locator 至少应与 `ChapterEvidenceAnchor` / projection report identity 相容，不相容时应 fail-closed 为 non-proof 或 E3。
- **实际行为**: 当前实现完全不读取 `ChapterFactInput.evidence_anchors` 的 metadata，也不对 `reference.document_year` 与 `projection.report_year` / anchor `document_year` 做一致性检查。只要 caller 能复用同一个 `anchor_id` 并让 excerpt 包含任一 material token，该 reference 就可以让该 fact 得到 `status="pass"`、`auditability_score=100`。
- **直接证据**: `confirm_chapter_evidence()` 在 line 168 只构造 `references_by_anchor`，未构造 anchor identity map；`confirm_projection_evidence()` 在 line 206 同样只按 reference anchor 分组；`_proof_references_for_fact()` 在 lines 339-344 只按 anchor id 取 proof；`_reference_is_proof()` 在 lines 360-365 只校验 reference 内部闭集配对；`_precision_issue()` 在 lines 385-386 对 `derived_calculation` 跳过 E1；`_matched_anchor_ids()` 在 lines 417-421 只用 token substring 决定 matched anchor。
- **影响**: Evidence Confirm 可输出错误的通过状态和 100 分 auditability score。后续若 report-level adoption 或 review workflow 消费该结果，会把错误年份、错误来源类型或 derived-only reference 误当成 same-anchor source support；当前测试只覆盖非法 `reference_kind/source_kind` 配对和跨 anchor，不覆盖“合法配对但与 fact anchor/report identity 不相容”的路径。
- **建议改法和验证点**: 在 public entry point 中从 `ChapterFactInput.evidence_anchors` 建立 `anchor_id -> ChapterEvidenceAnchor` map，并把 anchor metadata 传入 proof predicate；proof reference 必须引用存在于当前 chapter 的 anchor，且 `document_year/source_kind/section_id/table_id/row_locator` 与 anchor metadata 相容。至少新增两个回归：1. annual-report fact + same anchor id + `derived_calculation/derived` reference 不能关闭 E2/E3；2. same anchor id 但 `reference.document_year != projection.report_year` 或 != anchor `document_year` 时 fail-closed。
- **修复风险（低/中/高）**: 中。需要明确 phase 1 metadata compatibility 规则；若已有 tests 依赖宽松 reference metadata，需要同步收紧 fixture。
