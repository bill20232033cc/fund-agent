### F1-未修复-高-Plan permits fabricated section anchors for tables that have no section metadata

- **位置**: `Implementation Decisions` item 6 and item 8
- **问题类型**: 契约缺失 / 测试缺口 / 架构边界
- **当前写法**: The plan says to use `ParsedTable.section_id` if available, otherwise accept tables when the report contains `§4` and the table header/body shape matches the manager roster contract. It also requires anchors with `section_id="§4"`.
- **反例/失败场景**: Current `ParsedTable` has only `page_number`, `table_index`, `headers` and `rows`. A parsed annual report can contain `§4` text while a semantically similar table from another section is present in `report.tables`. Under the current plan, implementation could emit a direct `portfolio_manager_tenure_list.v1` value and label the table anchor as `§4` without proving the table belongs to `§4.1.2`.
- **为什么有问题**: The project requires same-source, traceable evidence. This gate's target is not just to satisfy the xfail fixture; it must avoid introducing a production extractor that fabricates row location. A header/body match alone proves table shape, not section ownership.
- **直接证据**: `fund_agent/fund/documents/models.py` defines `ParsedTable` with only `page_number`, `table_index`, `headers`, and `rows`. The plan's item 8 requires `section_id="§4"` anchors even when item 6 falls back to shape-only matching.
- **影响**: Implementation agent can make the five accepted tests pass while silently weakening evidence provenance in production-like inputs. Review cannot verify that direct rows are actually from `§4.1.2`, and later report/chapter consumers may treat fabricated anchors as annual-report evidence.
- **建议改法和验证点**:
  - Revise the plan to require a fail-closed section-title guard before direct extraction when table objects lack section metadata.
  - The minimum guard should require current report `§4` text to contain a manager-roster heading such as `4.1.2 基金经理简介` or an accepted equivalent, and the extracted table must match roster headers; if the heading is absent, return `missing` rather than direct.
  - Add a negative unit test where a manager-shaped table exists but `§4` lacks a manager roster heading; expected result is `missing`.
  - Keep the same-source test helper's `§4.1.2` heading as direct positive evidence.
- **修复风险（低/中/高）**: 中
- **严重程度（低/中/高/严重）**: 高
