# zhixing agent — 产品需求文档 (PRD)

> "知行合一" — 将投资认知转化为投资行动的 AI 伙伴

## 一、项目定位

### 1.1 一句话描述

zhixing agent 是一个**模板驱动的价值投资分析 AI Agent**，融合用户个人投资笔记（flomo/Obsidian/NoNotes）与公开财务数据，辅助个人价值投资者完成从标的筛选到投资后追踪的全流程。

### 1.2 与现有方案的差异

| 维度 | dayu-agent | 通用 AI (ChatGPT/DeepSeek) | zhixing agent |
|------|-----------|---------------------------|---------------|
| 数据源 | 公开财报（SEC/港交所） | 用户手动输入 | 公开财报 + **个人笔记** |
| 核心模式 | 财报解析 + 模板填充 | 自由对话 | **模板驱动 + 洞见注入** |
| 用户画像 | 买方分析师 | 所有人 | 个人价值投资者 |
| 独特价值 | 上万份财报的解析经验 | 灵活性 | **笔记洞见 → 投资决策** 闭环 |

### 1.3 核心理念

借鉴 dayu-agent 的**"按图索骥"**模式：

- **不是**让 AI 自由探索（"大海捞针"），容易产生幻觉和低质量输出
- **而是**用结构化的价值投资分析模板驱动 AI，在每个章节内精确提取和推理
- 模板是宿主（Host），LLM 在宿主约束下运行——"宿主强约束下的 LLM in the loop"

### 1.4 名字由来

"知行"取自王阳明"知行合一"——投资的认知（知）必须转化为投资行动（行），知而不行只是未知。

---

## 二、目标用户

**主要用户**：个人价值投资者，有以下特征：

- 使用笔记工具（flomo/Obsidian/NoNotes）记录投资思考和洞见
- 遵循价值投资理念（巴菲特/芒格/格雷厄姆体系）
- 需要结构化的分析框架，而非碎片化的 AI 问答
- 投资决策需要可追溯、可审计

---

## 三、核心功能

### 3.1 投资分析全流程（6 个场景）

```
Phase 0: 标的筛选 (Screening)
  → 量化指标过滤全市场，生成观察名单 (20-50 只)

Phase 1: 定性分析 (Qualitative)
  → 商业模式、护城河、管理层、行业地位、公司治理

Phase 2: 定量分析 (Quantitative)
  → 财务报表分析、关键比率、5-10 年趋势、同业对比

Phase 3: 估值定价 (Valuation)
  → DCF 估值、相对估值、安全边际、买入价确定

Phase 4: 投资决策 (Decision)
  → 仓位计算、买入策略、卖出条件、投资备忘录

Phase 5: 持仓追踪 (Tracking)
  → 季度检查、护城河监控、管理层红旗、调仓建议
```

### 3.2 个人笔记洞见提取

- 从 flomo/Obsidian/NoNotes 中提取投资相关洞见
- 将历史洞见注入对应的定性分析章节
- 每个洞见标注来源笔记，确保可追溯

### 3.3 模板驱动报告生成

- 5 份标准模板（定性/定量/估值/买入计划/追踪）
- 输出格式：Markdown → 可渲染为 HTML/PDF
- 每个章节的结论可追溯到数据源或笔记原文

---

## 四、输入源

### 4.1 用户笔记

| 平台 | 读取方式 | 难度 |
|------|---------|------|
| Obsidian | 直接读取本地 Vault 文件夹（纯 .md 文件） | 低 |
| flomo | HTML 导出 → Markdown 转换 → 解析 | 中 |
| NoNotes | 自有后端 API 直接读取 | 低（自有产品） |

### 4.2 财务数据

| 市场 | 数据源 | 覆盖 |
|------|--------|------|
| A 股 | AKShare（免费开源） | 财务报表、行情、估值指标 |
| 美股 | yfinance / Yahoo Finance | 财务报表、行情、估值指标 |
| 港股 | AKShare | 基础财务数据 |

### 4.3 未来可扩展

- dayu-agent 处理过的财报数据（作为更高质量的数据源）
- Tushare Pro（需要 token，数据更全）
- SEC EDGAR（美股深度分析）

---

## 五、输出

### 5.1 分析报告

完整的 Markdown 格式投资分析报告，包含所有模板章节，每个结论附带数据来源引用。

### 5.2 投资决策文档

结构化的买入/卖出计划，包含：
- 内在价值范围（熊/基准/牛三种情景）
- 安全边际和买入价格
- 仓位大小和建仓策略
- 卖出条件和追踪触发器

### 5.3 洞见关联图

用户笔记中与当前分析标的相关的所有历史洞见，按分析维度归类。

---

## 六、非目标（明确排除）

1. **不做通用 Agent** — 只服务于价值投资分析场景
2. **不做自动交易** — 输出分析和建议，不直接下单
3. **不做实时行情** — 周级别数据更新即可，不需要 tick 级
4. **不做社交功能** — MVP 阶段是个人工具，不是社区
5. **不做财报原文解析** — MVP 直接用 AKShare/yfinance 结构化数据，不碰 PDF/HTML 解析（这是 dayu-agent 的领域）

---

## 七、成功指标

| 指标 | MVP 目标 |
|------|---------|
| 对单标的完成端到端分析 | 覆盖 6 个场景全部章节 |
| 分析报告质量 | 用户（自己）认为比纯 ChatGPT 输出更有决策参考价值 |
| 笔记洞见注入 | 至少 3 条历史笔记洞见被融入分析 |
| 单次分析耗时 | < 5 分钟（CLI 模式） |
| 单次分析成本 | < ¥1（DeepSeek API） |
