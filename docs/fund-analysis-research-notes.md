## 二、基金年报结构研究

### 2.1 年报完整章节结构（2026 年新规）

| 章节 | 标题 | 对 Agent 的价值 |
|------|------|----------------|
| §1 | 重要提示及目录 | ⬜ 低 |
| §2 | 基金简介（含投资目标与策略） | 🟡 中——提取宣称策略，用于言行一致性检验 |
| §3 | 主要财务指标、净值表现及利润分配 | 🔴 高——R=A+B-C 的数据基础 |
| §4 | **管理人报告**（含投资策略和运作分析） | 🔴 **最高**——基金经理的"说" |
| §5 | 托管人报告 | ⬜ 低 |
| §6 | 审计报告 | ⬜ 低 |
| §7 | 财务会计报告（含附注） | 🟡 中——关联交易、费用明细 |
| §8 | **投资组合报告**（全部持仓+交易变动） | 🔴 **最高**——基金经理的"做" |
| §9 | **基金份额持有人情况**（含从业人员持有） | 🔴 高——利益一致性检验 |
| §10 | 份额变动 | 🟡 中——资金流向 |
| §11 | 重大事件揭示 | 🟡 中——人事变动、策略变更 |
| §12 | 备查文件目录 | ⬜ 低 |

### 2.2 年报的共性与不同

**共性**（所有基金年报都有的）：
- 章节结构完全一致（法规强制）
- 净值增长率 vs 业绩基准对比表格
- 前十大重仓股（季报也有）
- 费用数据
- 份额变动数据

**不同**（因基金类型和管理人而异）：
- §4 管理人报告的**深度和真诚度**差异巨大
- 投资策略阐述的**清晰度**差异巨大
- 业绩归因的**诚实度**差异巨大（跑输时是否坦诚解释）
- 换手率差异巨大（100% vs 2000%+）
- 基金经理自购金额差异巨大（0 vs 数百万）

### 2.3 费率和回报率之外：值得参考和事后验证的表述

| 表述类型 | 年报位置 | 事后验证方法 | Agent 可自动化程度 |
|---------|---------|-------------|------------------|
| **投资策略阐述** | §4 | 对比下一期实际持仓变化 | 🟡 半自动（需 LLM 提取+规则比对） |
| **后市展望/市场判断** | §4 | 对比下一期市场实际走势 | 🟡 半自动 |
| **业绩归因说明** | §4 | 对比 Brinson 归因数据 | 🔴 高（可量化验证） |
| **风险控制措施** | §4 | 对比实际最大回撤 | 🔴 高 |
| **行业配置逻辑** | §4 + §8 | 对比行业配置变化 | 🔴 高 |
| **基金经理自购** | §9 | 跨期追踪持有变化 | 🔴 高 |
| **换手率** | §8（2026 新规强制披露） | 跨期对比异常变化 | 🔴 高 |
| **关联交易/佣金分配** | §7 附注 | 审查合理性 | 🟡 半自动 |

### 2.4 关键发现：2026 年信息披露新规

2026 年 5 月 1 日起施行的新规带来几个对 Agent 有重大影响的变更：

1. **新增"盈利投资者数量占比"**——直接衡量投资者获得感
2. **新增"股票换手率"强制披露**——不再需要从交易数据推算
3. **中长期业绩延长至 7 年、10 年**——更多历史数据可用
4. **取消过去 1 个月业绩披露**——减少短期噪音

## 三、R = A + B - C 核心方法论

### 3.1 公式定义（来源：materials/2018 + E00/E02/E03）

```
投资者收益 R = Alpha (A) + Beta (B) - Cost (C)
```

| 组成 | 含义 | 确定性 | 获取方式 |
|------|------|--------|---------|
| **A** | 超额收益 | 低 | 主动择股+择时 |
| **B** | 市场平均收益 | 中 | 宽基指数基金 |
| **C** | 投资成本 | 高 | 显性+隐性费用 |

### 3.2 成本 C 的细分（关键！）

**看得见的费用**：申购费、赎回费
**看不见的费用**：管理费、托管费、基金交易成本、其他费用、销售服务费

> 实际案例：某主动型股票基金，管理费 1.20%/年 + 托管费 0.20%/年 + 交易成本 0.12%/年 + 其他 0.01%/年 = **每年 153 元/万元**

**隐性成本估算**：换手率 2000% 的基金，隐性交易成本可达 **9%-10%/年**

### 3.3 投资者行动优先级（有知有行的核心建议）

```
1. 首先控制成本 (C)  ← 确定性最高，省下的就是赚到的
2. 然后拿到 Beta (B) ← 长期确定性强
3. 最后追求 Alpha (A) ← 不确定性最高，需要足够的敬畏心
```

### 3.4 学术来源

陈鹏博士与耶鲁大学教授 Roger Ibbotson 2008 年合著论文《对冲基金的 ABC》。

### 3.5 Agent 实现要点

- **C 是最容易自动化计算的**：从年报直接读取费率，用换手率估算隐性成本
- **B 需要选择合适的基准**：不能简单用沪深 300，应根据基金实际风格设定（如有知有行为价值基金用中证 800 价值指数）
- **A = R - B - C**：扣除市场和成本后的剩余，才是基金经理真正的能力

## 四、基金体检四步走（有知有行实战框架）

### 4.1 四步框架

| 步骤 | 名称 | 核心问题 | 数据来源 |
|------|------|---------|---------|
| 1 | 看初心与诚意 | 基金经理跟你是不是一条心？ | §9 从业人员持有情况 |
| 2 | 看历史体验 | 投资者真的赚到钱了吗？ | §3 投资者收益率 vs 基金收益率 |
| 3 | 看能力与归因 | 超额收益从哪来？能持续吗？ | §3 + §4 + §8 |
| 4 | 看言行一致性 | 说的和做的一样吗？ | §4（说）vs §8（做） |

### 4.2 每步的关键发现

**第一步**：有知有行严选的硬性标准——**主动偏股基金经理必须持有自己管理的基金**。清仓行为需特别重视（可能预示离职）。

**第二步**：2025 年约 200 只主动偏股基金，投资者行为损失达 30% 以上。部分基金投资者行为损失达 70%-100%+。

> 终极目标不是挑出"业绩良好的基金"，而是挑出"能帮助投资者赚到钱的基金"。

**第三步**：超额收益需要拆分为仓位配置、行业配置、个股选择、股票交易四个来源。大成睿享 2025 年跑输基准是因为股票仓位仅 60%（基准 80%）。

**第四步**：换手率是关键指标。易方达逆向投资换手率从 2 倍提高到 4 倍，需关注。价值风格基金在 2025 年价值风格表现不佳时仍坚持自我，未切换到成长风格——这是**正面信号**。

## 五、有知有行的基金筛选框架（五层模型）

```
第一层：理念层
  R = A + B - C
  以风险控制为先，以风格稳定为重
  终极目标：选出"能帮助投资者赚到钱的基金"

第二层：分类层
  股票基金三维标签：市值（大/中/小/小微）× 风格（价值/均衡/成长）× 管理（主动/指数）
  债券基金四象限：短债/中长债 × 主动/指数

第三层：筛选层
  指数基金：规模、流动性、费率、跟踪误差
  主动基金：投资理念、公司文化、风格稳定性、利益绑定、知行一致性

第四层：分析层
  设定合理业绩基准（根据实际风格）
  R=A+B-C 收益归因（仓位、行业、个股、交易）
  抱团度分析
  风格稳定性追踪
  投资者收益 vs 基金收益

第五层：监控层
  基金经理持有变化
  换手率异常
  风格漂移
  基金经理变更
```

## 六、有知有行"投资第一课"核心方法论

> 来源：https://youzhiyouxing.cn/curriculum/lessons（17 节课，孟岩 & 张潇雨主讲）
> 与分析模板设计最相关的是：第 04-05 课（认知篇）、第 08 课（分散投资）、第 11 课（情绪篇）、第 12-16 课（实操篇）

### 6.1 "80 分投资者"的能力模型（导读）

格雷厄姆的名言定义了目标：

> "门外汉只需些许努力，便可以取得令人尊敬（即使并不可观）的结果。"

**"80 分"不需要**：选股、择时、看盘、K 线、高频交易
**"80 分"只需要**：理解收益来源 + 做好资产配置 + 选好资产 + 买好价格 + 长期持有

**对 Agent 的启示**：Agent 的目标用户就是"想成为 80 分投资者"的普通人，不是专业投资者。分析模板应该输出**简单明确的行动建议**，而非复杂的专业报告。

### 6.2 资产配置：四笔钱框架（第 12-13 课）

**核心理念**：投资业绩的影响因素排序——**投资者行为 > 资产配置 > 交易成本 > 选股/选基**

多数人过度关注"买什么"，忽略了"整体资产是否合理配置"。

**四笔钱分类**：

| 分类 | 用途 | 关键特征 | 适合的基金类型 |
|------|------|----------|--------------|
| 保险保障 | 人生托底 | 防守，避免被动卖出 | — |
| 活钱管理 | 随时要用的钱 | 流动性和安全性优先 | 货币基金 |
| 稳健理财 | 1-3 年内预期花销 | 比活钱收益更高 | 短债基金、中短债基金 |
| 长期投资 | 5 年以上不用的钱 | 可承受较高风险 | 宽基指数、主动权益 |

**对 Agent 的启示**：分析基金前，应先问"这笔钱属于四笔钱中的哪一笔？"不同类型的钱对应完全不同的基金选择标准。

### 6.3 股债配置与再平衡（第 13 课）

**格雷厄姆的经典框架**：
- 股债比例范围：25:75 ~ 75:25，多数人 50:50
- **再平衡是"投资里唯一免费的午餐"**

**A 股回测数据（2004-2020）**：

| 策略 | 年化收益 | 最大回撤 |
|------|----------|----------|
| 75:25（无再平衡） | 9.98% | 64% |
| 75:25（年度再平衡） | 11.38% | 53% |
| 50:50（年度再平衡） | ~9.6% | ~45% |
| 25:75（年度再平衡） | ~7.9% | 19% |

**预期收益推算公式**：
```
总收益 = 股票比例 × 8% + 债券比例 × 3% + 1%（再平衡增厚）
```

**四种提高收益的"武器"**：

| 武器 | 预期增厚 | 说明 |
|------|----------|------|
| 基于估值的低买高卖 | +3~5% | 温度计驱动 |
| 增强型指数基金 | +1~2% | 量化因子优化 |
| 主动型基金 | +2~3% | 优秀基金经理选股 |
| 债券型基金优化 | +1% | 通过杠杆放大收益 |

**对 Agent 的启示**：
- Agent 应该帮用户计算"我的钱应该怎么分"，而不只是"这只基金好不好"
- 再平衡提醒是高价值功能——偏离目标比例超过 10% 时触发
- 收益推算公式可以帮用户建立合理预期（年化 8%-12%，而非 20%+）

### 6.4 好资产选择：核心-卫星策略（第 14 课）

**"好资产"的定义**：对普通投资者来说，最重要的是投资一个**长期向上的市场**。

**为什么指数基金是好资产**：
- 优胜劣汰机制：指数定期调整，差的公司被淘汰
- 创业板指案例：初始 100 家公司中仅 18 家仍在指数中，持有指数年化 12.83% vs 持有初始成份股 7.77%
- A 股头部效应：前 15 家公司贡献了约 10 万亿财富

**并非所有指数都是好资产**：中证煤炭指数从 3300 点跌去 61%（2010-2020）。行业指数基金 ≠ 真正的指数化投资。

**核心-卫星策略**：

| 层级 | 标的 | 比例 | 作用 |
|------|------|------|------|
| **核心** | 沪深 300 + 中证 500 | 60%+ | 覆盖 A 股约 70% 市值 |
| **卫星** | 中概互联、消费、医药、红利等 | 灵活 | 降低波动，争取超额 |

**对 Agent 的启示**：
- 分析基金时，应评估它在用户组合中扮演"核心"还是"卫星"角色
- 行业主题基金应标注为"卫星"，提醒用户控制仓位
- 宽基指数基金应标注为"核心"，建议作为配置基础

### 6.5 好价格：温度计与收益归因（第 15 课）

**五条核心原则**：

| 原则 | 内容 | 数据支撑 |
|------|------|----------|
| 1. 尽可能一直待在市场中 | 错过涨幅最大的 5 天，年化从 12.9% 降至 10.1% | 万得全 A 15 年数据 |
| 2. 长期收益主要来自企业盈利增长 | 净资产增长 11.17% + 股息 1.52% + 估值变化 0.07% = 12.9% | 万得全 A |
| 3. 不必苛求买最低，但不要买太贵 | 买在最低点 10.69% vs 买在第一天 8.84% vs 买在最高点 5.63% | — |
| 4. 低估值买入可增厚收益 | 低估买入持有 5 年年化 > 10%，高估买入可能亏损 | 温度计回测 |
| 5. 高估卖出可大幅提高收益率 | 低买高卖 IRR 20% vs 普通定投约 8% | 2010-2015 回测 |

**约翰·博格的股市回报公式**：
```
股市回报 = 企业盈利增长 + 股息收益 + 估值变化
```
投资时间越长，估值变化贡献占比越低（万得全 A 15 年仅 0.07%）。

**三种定投策略对比**：

| 策略 | 内部收益率 |
|------|-----------|
| 普通定投 | ~8% |
| 低估加倍 + 高估不买 | 12% |
| 低估加倍 + 高估不买 + 高估卖出 | 20% |

**对 Agent 的启示**：
- 收益归因可以拆分为"盈利增长 + 股息 + 估值变化"，帮用户理解"我的钱是怎么赚到的"
- "尽可能待在市场中"——Agent 不应建议用户空仓等待
- 温度计驱动的定投调整（低估加倍/高估停止）是高价值功能

### 6.6 投资系统：12 个组成部分（第 16 课）

**马克·泰尔框架 + 有知有行的答案**：

| # | 组成部分 | 有知有行的答案 | Agent 可否自动化 |
|---|----------|---------------|----------------|
| 1 | 买什么 | 指数基金（+增强/主动） | ✅ 基金筛选 |
| 2 | 什么时候买 | 按月检视，定期购买 | ✅ 定投提醒 |
| 3 | 买价 | 根据温度计，低估多买 | ✅ 温度计+仓位建议 |
| 4 | 如何买 | 现金 | ⬜ — |
| 5 | 买入比例 | 分散投入多只指数 | ✅ 组合配置建议 |
| 6 | 投资过程监控 | 指数基金自我管理 | ✅ 年报/换手率监控 |
| 7 | 何时卖 | 温度高估时 / 再平衡时 | ✅ 卖出信号 |
| 8 | 组合结构和杠杆 | 不用杠杆 | ✅ 杠杆检测 |

**对 Agent 的启示**：这 12 个问题就是 Agent 功能的**完整需求清单**。其中 8 项可以自动化，4 项需要用户自行决策。

### 6.7 市场周期与预期收益率（第 04-05 课）

**A 股预期收益率推导（自上而下）**：

| 层级 | 指标 | 数值 |
|------|------|------|
| 基础层 | 中国未来 GDP 年均增速 | 5%-6% |
| 上市公司溢价 | 优于全部企业 | +2-4 百分点 |
| **基础预期收益率** | **A 股长期年化** | **8%-10%** |

**提升路径**：精选优质公司（+2-3%）→ 纳入境外优质公司 → 利用市场波动低买高卖（+3-5%）
**终极区间**：8%-15%（已是巴菲特级别，巴菲特真实回报率剔除杠杆后约 12.5%）

**市场周期的三层模型**：
```
经济周期（最内层，波动最小，GDP 通常 2%-3%）
  └─ 企业盈利周期（中间层，经营杠杆+财务杠杆放大）
       └─ 情绪/市场周期（最外层，波动最大，贪婪与恐惧）
```

**关键数据**：1970-2016 年美国股市平均回报 10%，但 47 年中只有 **3 年**回报落在 8%-12% 区间——"常态其实是非常态"。

### 6.8 分散投资的具体建议（第 08 课）

**真正分散的三个维度**：

| 维度 | 含义 | 反面案例 |
|------|------|---------|
| **资产层面** | 股票、债券、黄金等不同资产类别 | 买了多个平台的 P2P（假分散） |
| **行业层面** | 不同行业之间分散 | 全部投资餐饮公司 |
| **地理层面** | 跨越不同国家和市场 | 本土偏见（Home Bias） |

**加拿大养老基金（CPP）最佳实践**：1999 年 100% 政府债券 → 2019 年股权 54.9% + 实体资产 20%+ + 加拿大股票仅 15.6%（全球分散）。

### 6.9 投资者行为偏差与干预设计（第 11 课）

**四种关键偏差及干预策略**：

| 偏差类型 | 表现 | 干预策略 | Agent 实现 |
|---------|------|---------|-----------|
| **损失厌恶** | 平安跌 4% 就恐慌"20 万还剩 10 万吗" | 信息隔离：减少查看频率 | 定期报告（非实时推送） |
| **短期主义** | 无法忍受短期亏损，即使长期正确 | 信任委托：交给系统执行 | 冷静期机制 |
| **计划谬误** | 高估自己的意志力 | 物理隔离：增加操作摩擦力 | 买入前的强制等待 |
| **频繁查看** | 每天看行情放大情绪波动 | 预算隔离：小额"赌博账户" | 仓位上限控制 |

**孟岩帮母亲管理账户的案例**：修改密码 + 不讨论具体操作 + 每年仅报告一次净值 → 8 年后资产变为原来的 **4 倍**。

> "投资成功，是我们变成一个更好的人之后自然的结果。"

### 7.5 投资者行为偏差的量化数据（核心！）

**来源**：[公募权益类基金投资者盈利洞察报告](https://youzhiyouxing.cn/materials/1036)（景顺/富国/交银，129 只基金、4682 万客户、5.65 亿笔交易）

**核心发现**：
```
基民投基收益 = 基金损益 + 基民行为损益
基民的"操作"给收益带来的损耗率接近 -60%
```

**持仓时长与收益关系**：

| 持仓时长 | 平均收益率 | 盈利人数占比 |
|---------|-----------|------------|
| <3 个月 | 负收益 | 39.10% |
| >3 个月 | 由负转正 | 63.72% |
| >10 年 | **117.38%** | **98.41%** |

**但**：45.96% 客户持仓 <3 个月，仅 1.73% 客户持有 >10 年

**其他关键数据**：
- 女性 vs 男性平均收益率差：女性高 **1.57** 个百分点（交易频率更低）
- 定投 vs 非定投平均收益率差：定投高 **2.94** 个百分点
- 60 岁以上 vs 30 岁以下：19.05% vs <3%（年龄越大越能拿住）
- 33.39% 在盈利不超 10% 时赎回（最高比例——过早止盈）

### 8.3 关键发现与启发

#### 发现 1：指数温度表是天然的"核心-卫星"配置引擎

温度数据直接告诉你哪些资产便宜、哪些贵：

```
当前建议（基于温度）：
核心仓位 → 中估宽基：沪深300(63°)、上证50(55°)
卫星加仓 → 低估方向：800消费(0°)、全指医药(3°)、中证红利(27°)
回避/减仓 → 高估方向：全指信息(93°)、中证500(77°)、创业板指(75°)
```

#### 发现 2：内在收益率是比 PE/PB 更直观的估值锚点

```
预期长期年化回报 ≈ 内在收益率 + 股息率

关键对比：
- 中证红利：4.37% + 4.87% = 9.24%（低估+高股息，总回报预期优秀）

```

→ 温度高不一定贵，需要结合内在收益率判断。创业板指 75° 看似高估，但内在收益率 8.66% 说明成长性支撑了估值。

#### 发现 3：债市温度 79° = 债券配置需谨慎

- 债市高估意味着新买入的债券票息低（10 年期国债仅 1.76%）
- 但债券的核心作用是**降低组合波动**，不只是收益
- **建议**：债市高估时优先短债基金，减少中长债久期暴露

#### 发现 4：指数详情页的三个被低估的分析维度

1. **温度走势图**：可视化温度变化趋势，可判断当前处于历史什么位置
2. **成长分析**（季度更新）：净利润增速 vs 温度的交叉验证
   - 温度高 + 净利润增速高 → 可能不算贵
   - 温度高 + 净利润增速低 → 真正高估
3. **行业分布**：前两大行业占比 >30% 需标注集中度风险

| 数据类型 | 更新频率 | Agent 用途 |

| 全市场/指数温度 | 每晚 20:00 | 日频信号，驱动买卖决策 |
| 中国互联网指数 | 下一交易日 | 海外时差延迟 |
| 指数成长分析 | 季度 | 低频信号，基本面验证 |
| 中证 2000 内在收益率 | 暂无 | 2023 年 8 月发布，数据不完整 |

## 九、从第一性原理出发的思考与质疑

### 9.1 基金投资的本质

基金投资本质是**委托代理关系**——你把钱交给基金经理，期望他通过专业能力实现风险调整后的超额收益。

核心问题只有三个：
1. **有没有能力？**（Alpha 是否真实存在）
2. **会不会好好干？**（利益是否对齐）
3. **能不能持续？**（知行合一、风格稳定）

### 9.2 对现有方案的质疑与建议

#### ✅ 值得深入研究的

| 方向 | 理由 |
|------|------|
| **R=A+B-C 收益归因** | 有知有行已验证的核心框架，数据可从年报获取，计算确定性强 |
| **基金经理自购（利益绑定）** | 年报直接披露，零成本获取，强信号 |
| **投资者收益率 vs 基金收益率** | 2026 新规强制披露，直接衡量"基金好不好用" |
| **换手率 → 隐性成本** | 2026 新规强制披露，计算简单，意义重大 |
| **风格稳定性（言行一致性）** | §4 vs §8 交叉验证，是判断基金经理质量的核心 |
| **同类基金费率对比** | 确定性高，长期复利效应大 |

#### ⚠️ 可以研究但需控制投入的

| 方向 | 注意事项 |
|------|---------|
| **横向比较（Phase 2）** | 需要大量同类基金数据，数据获取成本高；建议先用有知有行严选基金作为"已筛选池"，在其内部做比较 |
| **市场温度计（Phase 3）** | 有知有行温度计的核心算法是商业机密，我们只能爬取温度值而非自己计算；作为参考信号即可，不必自己造温度计 |
| **Brinson 归因（仓位/行业/个股拆分）** | 理论优美但需要大量持仓数据（全部持仓，非仅前十大），且计算复杂；建议简化为"超额收益是否为正 + 是否持续" |

#### ❌ 建议不做的（或大幅简化的）

| 方向 | 理由 |
|------|------|
| **自己构建温度计** | 核心算法是商业机密，自己造不如直接爬有知有行的数据 |
| **复杂的行为偏差检测** | 需要用户的完整交易记录，数据获取难度大，且用户可能不愿提供 |
| **LLM 解读基金经理"话术"** | 基金经理的管理人报告充满套话，LLM 解读的信噪比低；不如直接看"做"（持仓数据） |
| **多因子选基模型** | 过度工程化，Fama & French 研究表明仅 2%-3% 的基金经理有统计上显著的超额收益，复杂的模型可能只是在拟合噪声 |
| **实时监控和自动调仓** | 监管风险 + 个人使用场景不需要高频交易 |

### 9.3 核心洞察：Agent 的真正价值不在"分析"而在"干预"

从第一性原理出发，基金分析的难点不在于"算出哪个基金好"（有知有行已经做了），而在于：

1. **在用户准备追涨时拦截他**——这是有知有行没做的
2. **在用户持有亏损基金时告诉他"该不该割"**——这也是有知有行没做的
3. **把年报中隐藏的关键数据（换手率、隐性成本、自购变化）自动提取出来**——降低信息不透明

**Agent 的差异化价值 = 自动化信息提取 + 个性化行为干预**

### 9.4 融合课程方法论后的分析模板（v2）

基于以上研究，建议 Agent 的分析模板聚焦以下**高确定性、高价值**的分析项：

```
基金分析报告（v2 —— 融合投资第一课方法论）
│
├── 前置：资金属性判断
│   - 这笔钱属于四笔钱中的哪一笔？（活钱/稳健/长期）
│   - 投资期限：1 年内 / 1-3 年 / 3-5 年 / 5 年以上
│   → 不同资金属性 → 完全不同的基金选择标准
│
├── 1. 基金画像
│   - 类型、规模、经理、费率
│   - 角色：核心（宽基指数）还是卫星（行业/主题）？
│   - 好资产检验：该指数/基金是否代表长期向上的市场？
│
├── 2. 成本分析（C）—— 确定性最高，最先看
│   - 显性成本：管理费 + 托管费 + 申赎费
│   - 隐性成本：换手率 × 0.3%
│   - 总成本 vs 同类平均
│   - 成本是否侵蚀了超额收益？（C > A → 红灯）
│
├── 3. 收益归因（R = A + B - C）
│   - B：基准收益（根据风格选择合适基准）
│   - A：超额收益 = 基金收益 - 基准收益
│   - 博格公式拆分：收益 = 企业盈利增长 + 股息 + 估值变化
│   - A 是否为正？是否持续？是否覆盖 C？
│   - 预期收益率是否合理？（A 股长期 8%-10%，不要幻想 20%+）
│
├── 4. 利益一致性
│   - 基金经理是否自购？（是/否/金额）
│   - 自购变化趋势（加仓/减仓/清仓 → 离职预警）
│   - 投资者收益率 vs 基金收益率差距
│   → 终极目标：选出"能帮助投资者赚到钱的基金"
│
├── 5. 风格稳定性（言行一致性）
│   - §4（说）vs §8（做）交叉验证
│   - 实际持仓风格 vs 宣称风格（偏离度）
│   - 换手率变化趋势
│   - 在风格不利时是否坚持？（如 2025 年价值基金不切换成长 → 正面信号）
│
├── 6. 市场时机（好价格）
│   - 当前温度：低估 / 中估 / 高估
│   - 买入建议：低估加倍 / 中估正常 / 高估停止
│   - 原则：尽可能待在市场中，不要空仓等待
│
├── 7. 组合视角（好配置）
│   - 该基金在组合中的角色（核心/卫星）
│   - 当前股债比例 vs 目标比例
│   - 是否需要再平衡？（偏离 > 10% 触发）
│   - 分散度检查：资产/行业/地理三维度
│
└── 8. 行为干预（Agent 差异化价值）
    - 冷静期：温度 > 85° 且权益 > 80% → 建议观望
    - 仓位上限：单只基金不超过组合的 20%
    - 追涨检测：是否在净值高位买入？
    - 预期管理：年化 8%-12% 是合理的，不要追涨杀跌
```

## 十一、温度计开源项目研究

> 研究目的：评估自建温度计的可行性，寻找最佳实践参考

### 11.1 核心发现

**温度计的算法并不复杂**——4 行 Python 就能实现基础版本：

```python
import akshare as ak
# 获取 A 股全市场 PE 中位数
pe_df = ak.stock_a_ttm_lyr()
# 计算历史百分位（这就是温度计的核心）
pe_percentile = pe_df['middlePETTM'].rank(pct=True) * 100
```

**有知有行的商业机密主要是权重调优和清洗规则**，但基础版本完全可以自己构建。

### 11.2 开源项目对比

| 项目 | 推荐度 | 核心特性 | 适用场景 |
|------|--------|---------|---------|
| **[myquant](https://gitee.com/dududuuu/myquant)** | ★★★★★ | 等权 PE/PB 百分位，支持全市场+单指数，plotly 可视化，MIT 协议 | **直接 fork 使用** |

| **Fin-Maestro** | ★★★☆☆ | 指数健康度检查模块，红绿灯颜色编码 | 架构参考（印度市场） |
| **CSDN 四行代码** | ★★★☆☆ | 最简原型 | 快速验证 |

|------|-------------|---------|

| **集思录** | 完全公开 | PB/PE 中位数，扣除停牌/次新/ST，1994 年起始 |

### 11.4 自建温度计建议

**技术路径**：
1. 数据源：AKShare `stock_a_ttm_lyr()` / `stock_a_all_pb()`
2. 核心算法：`rank(pct=True) * 100` 计算历史百分位
3. 数据清洗：扣除停牌股、1 年内新股、ST 股票（参考集思录）
4. 权重选择：等权（有知有行方式）vs 市值加权
5. 考察周期：至少 10 年（两轮牛熊）
6. 可视化：plotly 绘制温度走势图

**与有知有行交叉验证**：
- 自建温度计作为参考，与有知有行温度对比
- 差异大时分析原因（权重、清洗规则、周期选择）

**状态**：保留为研究方向，MVP 先用有知有行数据。

## 十二、行为偏差检测方案修订

### 12.1 原方案（v3.x）的问题

原方案设计了复杂的实时行为偏差检测机制：

| 偏差类型 | 原方案干预方式 | 实现难点 |
|---------|-------------|---------|
| 损失厌恶 | 减少查看频率 | Agent 无法控制用户行为 |
| 短期主义 | 冷静期机制 | 需要用户主动触发 |
| 计划谬误 | 买入前强制等待 | 需要用户主动触发 |
| 频繁查看 | 仓位上限控制 | 需要用户持仓数据 |

**核心问题**：
1. Agent 是**被动触发**的——用户不问就不答
2. 需要**用户交易记录**——获取难度大
3. "检测追涨"需要知道用户的**买入价格和时机**
4. 干预只能"建议"不能"阻止"

### 12.2 修订方案：买入前检查清单

将"自动检测"改为**"用户主动触发 + Agent 被动响应"**模式：

**触发场景**：用户说"我想买 XX 基金"

**Agent 响应**：5 个问题检查清单

| # | 问题 | 红灯条件 | 黄灯条件 | 绿灯条件 |
|---|------|---------|---------|---------|
| 1 | 资金属性 | — | — | 明确属于四笔钱中的哪一笔 |
| 2 | 市场温度 | 全市场 >85° 且权益 >80% | 全市场 70°-85° | 全市场 <70° |
| 3 | 指数估值 | 跟踪指数温度 >90° | 跟踪指数温度 70°-90° | 跟踪指数温度 <70° |
| 4 | 费率水平 | 总成本 >2%/年 | 总成本 1%-2%/年 | 总成本 <1%/年 |
| 5 | 仓位占比 | 单只基金 >20% 组合 | 单只基金 10%-20% | 单只基金 <10% |

**输出**：
- 🔴 红灯（≥2 个）：建议暂缓，提供替代方案
- 🟡 黄灯（1 个）：可以买入，但需注意风险提示
- 🟢 绿灯（0 个）：建议买入

**优势**：
- 不需要用户交易记录
- 被动响应模式，用户主动触发
- 规则透明，用户可理解
- 可快速实现

### 12.3 定期体检报告（补充）

**模式**：用户主动上传持仓 Excel 或逐笔录入 → Agent 每周/每月生成一页纸体检报告

**报告内容**：
- 组合股债比例 vs 目标比例
- 偏离度 >10% 的基金列表
- 温度高估的基金提示
- 再平衡建议（优先使用新增资金）

**优势**：
- 用户控制数据输入
- Agent 主动推送（定时任务）
- 一页纸，不打扰

### 12.4 砍掉的功能

| 功能 | 砍掉原因 |
|------|---------|
| 实时追涨检测 | 需要交易记录，获取不可行 |
| 自动行为分析 | 需要用户授权获取账户数据，合规风险高 |
| 复杂干预规则 | 只能建议不能阻止，ROI 低 |

**总结**：行为干预的理念是对的，但实现路径从"自动检测"改为"买入前检查清单 + 定期体检报告"，更务实、更可落地。

## 十三、审计机制设计要点（借鉴 Dayu-Agent）

> **日期**: 2026-05-15
> **来源**: Dayu-Agent 审计能力深度分析文档

### 13.1 核心理念

**让 AI 分析基金的方式从"大海捞针"变成"按图索骥"，让数据有置信度，让投资结论可审计、可追踪。**

### 13.2 基金 Agent 与公司分析 Agent 的差异

| 特性 | 公司财报分析 | 基金年报分析 |
|------|-------------|-------------|
| 叙事性 | 高（MD&A 数千字） | 低（§4 管理人报告仅数百字） |
| 数据结构 | 复杂（三大表+附注） | 简单（净值+持仓+费用） |
| LLM 依赖 | 高（需理解管理层叙事） | 低（主要是结构化数据提取） |
| 幻觉风险 | 高（LLM 编造业务细节） | 中（主要是数据锚点精确性问题） |
| 审计重点 | 证据锚点精确性 | 计算正确性 + 证据锚点 |

### 13.3 三层审计架构

```
程序审计（Programmatic）→ LLM 审计 → 证据复核（Evidence Confirm）
```

**基金 Agent 的特点**：程序审计占比更高（~60%），因为计算验证多。

### 13.4 审计规则体系（基金专用）

在 Dayu-Agent 的 P/E/C/S 规则基础上，新增：

| 类别 | 规则码 | 含义 | 基金场景示例 |
|------|--------|------|-------------|
| **计算类(L)** | L1 | R=A+B-C 计算错误 | 数值计算不闭合 |
| | L2 | 百分位计算错误 | 同类排名百分位计算有误 |
| **规则类(R)** | R1 | 检查清单规则应用错误 | 温度 85° 应触发冷静期但未触发 |
| | R2 | 决策与评分不一致 | 评分 4.5 却给出绿灯 |

### 13.5 修复策略差异

| Dayu-Agent | Fund-Agent |
|------------|------------|
| patch 为主 | recalculate + patch 为主 |
| 证据锚点重写 | 重新计算 + 证据锚点重写 |

### 13.6 证据锚点格式（基金专用）

| 数据类型 | 锚点格式 | 示例 |
|---------|---------|------|
| 年报数据 | `年报{年份}§{章节}表{编号}行{行号}` | `年报2025§3表2行3` |
| 招募说明书 | `招募说明书第{页}页` | `招募说明书第15页` |
| 温度计数据 | `温度计{日期}` | `温度计2026-05-14` |
| 净值数据 | `净值{日期}来源{平台}` | `净值2026-05-14来源天天基金` |
| 计算结果 | `计算:{公式}输入:{来源}` | `计算:A+B-C输入:年报2025§3` |

### 13.7 开发优先级

| 阶段 | 审计功能 | 优先级 |
|------|---------|--------|
| MVP | 程序审计（P1/P2/L1/R1） | 🔴 P0 |
| v2 | 证据锚点规范 | 🔴 P0 |
| v2 | LLM 审计（E1/E2/C1） | 🟡 P1 |
| v3 | 证据复核 + 锚点重写 | 🟡 P1 |

### 14.1 E大投资体系总览

E大的投资体系由两大子系统构成：

```
投资体系
├── 观察系统（世界观）
│   ├── 估值体系（PE/PB 历史百分位）
│   ├── 资金面分析

│
└── 操作系统（方法论）

    ├── 中短期策略：网格策略（1.0/2.0/3.0）
    └── 辅助策略：资产配置 + 再平衡
```

**核心金句**：
> "投资是科学，也是艺术，但你首先要懂科学。"
> "小孩子才选择，成年人都要。"（长期价值 + 短期交易价值都要）
> "波动不必然是风险，买在波峰才是风险。"

### 14.2 四大核心策略详解

#### 14.2.1 定期不定额策略

**本质**：在定期定额基础上，根据估值动态调整投入金额。

| 估值水平 | 操作 | 金额 |
|---------|------|------|
| 正常估值 | 正常买入 | 1 份 |
| 低估值 | 加倍买入 | 2 份 |
| 极低估值（钻石坑） | 大幅加码 | 4-8 份 |
| 高估值 | 停止买入 | 0 份 |
| 极高估值 | 卖出 | - |

**关键规则**：
- 每月最多操作一次，一年不超过 12 次
- 存量资金分成 **150 份**（历史最长熊市买入 144 份的经验）
- 每月结余必须放入投资账户作为后备军
- "每 3 年会出现一次买入 4 份的机会，每 5 年会出现一次买入 8 份的机会"

**可借鉴点**：
- ✅ 基金 Agent 的"买入前检查清单"可融入估值判断维度
- ✅ 150 份的资金管理思路可用于组合管理的仓位规划
- ✅ "超额收益绝大部分产生在钻石坑"的理念可用于市场时机模块

#### 14.2.2 目标市值策略

**本质**：设定一个目标市值，价格下跌时买入更多份额以维持目标，上涨时卖出多余份额。

**与定期不定额的区别**：
- 定期不定额：**定期**投入，金额随估值变化
- 目标市值：**不定期**操作，由价格偏离触发

**适用场景**：
- 钻石坑里大量买入低估指数，使其达到目标仓位
- 长期持有品种的动态再平衡

**可借鉴点**：
- ✅ 组合管理模块可引入"目标市值"作为再平衡触发条件
- ✅ 与温度计结合：温度越低，目标市值中权益占比越高

#### 14.2.3 网格策略（1.0 → 2.0 → 3.0）

**核心思想**：在震荡市中，通过机械化的低买高卖获取波动利润。

**1.0 基础版**：
- 选品种（必须是不死的品种，即指数基金）
- 列网格表格（价格、金额、日期）
- 做压力测试（**最重要！**）
- 设置交易提醒
- 按提醒执行

**2.0 进阶版（三个子策略）**：

| 子策略 | 名称 | 解决的问题 | 操作方式 |
|--------|------|-----------|---------|
| 2.1 | 留利润 | 单边上涨赚得不够多 | 每次卖出时留下利润份额（0 成本）永不卖出 |
| 2.2 | 逐格加码 | 底部利润过低 | 越跌买越多，每格比上一格加 5%-10% |
| 2.3 | 一网打尽 | 波动幅度不确定 | 同时设置 5%/15%/30% 三层网格 |

**网格策略的关键约束**：
1. **品种必须不死**（指数基金，绝不用于个股）
2. **压力测试最重要**（模拟最大下跌 40%-60%，确认能扛住）
3. **开网时机**：价格略低于价值时
4. **网格大小**：普通品种 5%，波动大的 10%
5. **相关性高的品种不要重复开**
6. **网格是配角**，长期策略才是主力

**可借鉴点**：
- ✅ "压力测试"理念可融入基金分析模板——分析基金时模拟最大回撤
- ✅ "品种必须不死"可作为基金筛选的硬性门槛（排除可能清盘的基金）
- ✅ 网格策略的"留利润"子策略可用于组合管理的止盈设计
- ⚠️ 网格策略本身不适合作为 Agent 的核心功能，但可作为高级功能

#### 14.2.4 资产配置与再平衡

**E大的资产配置思路**：
- 大类资产配置只能自己做，没人能替你
- 资产之间的**相关性**是关键（低相关性才能分散风险）
- 定期或定幅再平衡（3-6 个月，或某个品种涨 50% 后触发）

**E大推荐的简单策略**：
> 25% 沪深300 + 25% 中证500 + 50% 利率债，定期再平衡。
> "别看简单，秒杀 60% 的 A股投资者毫无问题。"

**可借鉴点**：
- ✅ 组合管理模块的"六维健康诊断"可增加"资产相关性"维度
- ✅ 再平衡触发条件可参考 E大的"定期 + 定幅"双触发机制

### 14.5 对基金 Agent 分析模板的具体改进建议

#### 14.5.1 买入前检查清单增强

在现有的 5 问检查清单基础上，增加 E大视角的维度：

| 维度 | 检查项 | 来源 | 判定规则 |
|------|--------|------|---------|
| 估值 | 当前估值处于历史什么位置？ | E大估值体系 | PE/PB 百分位 < 30% → 🟢 |
| 成本 | 买入成本是否控制在相对底部？ | E大长赢计划 | 距离近 3 年估值低点 < 20% → 🟢 |
| 压力测试 | 最大回撤你能扛住吗？ | E大网格策略 | 模拟 -40% 回撤，确认可接受 |
| 退出计划 | 你是否已制定卖出预案？ | E大纪律投资 | 有明确的止盈/止损条件 → 🟢 |
| 资金属性 | 这笔钱 3-4 年内不会用？ | E大长短线比例 | 是 → 🟢 |

#### 14.5.2 基金分析模板增强

在"年报 X 光机"中增加 E大视角的分析维度：

| 分析维度 | 具体内容 | 数据来源 |
|---------|---------|---------|
| 基金"不死性"评估 | 基金规模、成立年限、公司实力 | 天天基金/晨星 |
| 压力测试 | 模拟 -30%/-40%/-50% 场景下的回撤 | 历史最大回撤数据 |
| 持仓相关性 | 与投资者已有持仓的相关性 | 用户上传持仓数据 |
| 波动特征 | 年化波动率、最大回撤、夏普比率 | 年报/天天基金 |

#### 14.5.3 组合管理增强

| 功能 | E大理念 | 实现方式 |
|------|--------|---------|
| 150 份仓位规划 | 存量资金分成 150 份 | 用户输入总资金 → 自动计算每份金额 |
| 长短期仓位比例 | 3-4 年不用的钱做长线，1-2 年不用的做波段 | 用户输入资金使用计划 → 建议比例 |
| 再平衡触发 | 定期（3-6 月）+ 定幅（涨 50%） | 定期提醒 + 阈值监控 |
| 目标市值 | 设定权益/债券目标比例 | 偏离超过 5% 时提醒再平衡 |
